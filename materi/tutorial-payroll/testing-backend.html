🧪 Testing Backend dengan Tinker & Postman

🚀 Tutorial Proyek ⏱️ 45 menit 🔬 Praktik Testing
⚙️ Versi: 🔥 Laravel 13.x ⚡ Livewire 3.x 🐘 PHP 8.3+ 📬 Postman 🛡️ Laravel Sanctum Testing ini dilakukan setelah Tahap 4 selesai — semua model (Employee & Payroll) dan logika kalkulasi sudah ada, sehingga testing bisa end-to-end. Sanctum sudah include di Laravel 13, tidak perlu install terpisah.

📌 Prasyarat

  • Model Employee + tabel employees sudah dibuat (Tahap 3).
  • Model Payroll + tabel payrolls + logika kalkulasi sudah dibuat (Tahap 4).
  • Database sudah terkoneksi dan php artisan migrate sudah dijalankan.
  • Postman sudah terinstall — unduh gratis di postman.com/downloads.

🎯 Tujuan Belajar

Setelah menyelesaikan materi ini, kamu akan mampu:

  • Menguji logika Model Eloquent langsung dari terminal dengan Laravel Tinker
  • Membuat API endpoint sederhana menggunakan routes/api.php
  • Menguji setiap endpoint (GET, POST, PUT, DELETE) menggunakan Postman
  • Memahami konsep API token dengan Laravel Sanctum
  • Memverifikasi respons JSON sebelum frontend dibangun

🤔 Kenapa Testing Backend Sebelum Frontend?

Banyak pemula langsung membangun tampilan Livewire (frontend) tanpa memastikan logika backend sudah benar. Ini menyebabkan debugging yang membingungkan — error bisa datang dari backend, frontend, atau keduanya sekaligus.

❌ Tanpa Testing Backend

  • Buat Livewire, klik Simpan → tidak ada yang terjadi
  • Entah salah di form, validasi, atau query?
  • Debugging 3 layer sekaligus: HTML, PHP, DB
  • Waktu debugging = 3× waktu coding

✅ Dengan Testing Backend Dulu

  • Test API → data masuk ke DB ✓
  • Test validasi → error response sesuai ✓
  • Saat pasang Livewire, tinggal sambungkan
  • Kalau ada bug, pasti di frontend saja
🏭

Ini Cara Kerja Developer Profesional

Di perusahaan, backend developer dan frontend developer bekerja terpisah. Backend dev membangun dan mendokumentasikan API — frontend dev memanggil API itu. Testing dengan Postman adalah cara backend dev memverifikasi pekerjaannya sebelum diserahkan ke tim frontend.

🖥️ Bagian 1: Laravel Tinker — Test Model di Terminal

Sebelum membuat API, kita bisa langsung test model Eloquent dari terminal menggunakan Tinker — PHP REPL (Read-Eval-Print Loop) bawaan Laravel. Tidak perlu browser, tidak perlu Postman dulu.

Terminal — masuk ke Tinker 
php artisan tinker

Setelah masuk Tinker, ketik perintah-perintah berikut satu per satu:

Test 1: Create Employee

Tinker 
use App\Models\Employee;

// Buat karyawan baru
Employee::create([
    'nik'      => 'EMP-001',
    'name'     => 'Budi Santoso',
    'position' => 'Staff IT',
]);

// Output yang diharapkan: objek Employee dengan id, nik, name, position, timestamps

Test 2: Read & Query

Tinker 
// Ambil semua karyawan
Employee::all();

// Filter berdasarkan jabatan
Employee::where('position', 'Staff IT')->get();

// Cari berdasarkan NIK
Employee::where('nik', 'EMP-001')->first();

// Hitung total karyawan
Employee::count();

Test 3: Update & Delete

Tinker

Employee::where('nik', 'EMP-001')->update(['position' => 'Senior IT']);

// Delete karyawan
Employee::where('nik', 'EMP-001')->delete();

📬 Bagian 2: Setup API dengan Laravel Sanctum

Tinker bagus untuk testing cepat, tapi Postman lebih realistis — kita bisa test request HTTP persis seperti yang nanti akan dikirim dari aplikasi nyata. Untuk ini, kita perlu membuat API endpoint dan autentikasi token.

Langkah 1: Aktifkan API di Laravel

Laravel 13 sudah include Sanctum, tapi perlu diaktifkan dengan satu perintah:

Terminal 
# Aktifkan API routing + Sanctum (hanya perlu sekali)
php artisan install:api

Perintah ini akan membuat file routes/api.php dan mengkonfigurasi Sanctum secara otomatis.

Langkah 2: Tambahkan HasApiTokens ke User Model

Ini diperlukan agar kita bisa membuat token dari Tinker untuk dipakai di Postman.

app/Models/User.php 
use Laravel\Sanctum\HasApiTokens;  // ← tambahkan baris ini

class User extends Authenticatable
{
    use HasApiTokens, HasFactory, Notifiable;  // ← tambahkan HasApiTokens
    // ... sisa kode tetap sama
}

Langkah 2b: Ambil Token via Tinker

Login/register sudah ditangani Breeze di halaman web. Untuk Postman, cukup buat token langsung dari terminal:

Terminal — Tinker 
php artisan tinker

// Ambil user admin yang sudah ada
$user = App\Models\User::where('email', 'admin@payroll.test')->first();

// Buat token untuk sesi Postman
$token = $user->createToken('postman-test')->plainTextToken;

// Tampilkan tokennya — copy nilai ini!
echo $token;
📋

Simpan token ini di Postman Environment

Copy output echo $token, pergi ke Environments → Payroll Dev, isi variable TOKEN dengan nilai tersebut. Semua request Postman selanjutnya akan otomatis menggunakan token ini.

Langkah 3: Buat API Controller untuk Employee

Terminal 
# Buat API controller (--api = tanpa create/edit form, --resource = semua method CRUD)
php artisan make:controller Api/EmployeeController --api
php artisan make:controller Api/PayrollController --api

Isi app/Http/Controllers/Api/EmployeeController.php:

app/Http/Controllers/Api/EmployeeController.php 
namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Models\Employee;
use Illuminate\Http\Request;

class EmployeeController extends Controller
{
    // GET /api/employees
    public function index()
    {
        return Employee::orderBy('name')->get();
    }

    // POST /api/employees
    public function store(Request $request)
    {
        $data = $request->validate([
            'nik'      => 'required|unique:employees,nik',
            'name'     => 'required|min:3',
            'position' => 'required',
        ]);

        $employee = Employee::create($data);

        return response()->json($employee, 201);
    }

    // GET /api/employees/{id}
    public function show(Employee $employee)
    {
        return $employee;
    }

    // PUT /api/employees/{id}
    public function update(Request $request, Employee $employee)
    {
        $data = $request->validate([
            'nik'      => 'required|unique:employees,nik,' . $employee->id,
            'name'     => 'required|min:3',
            'position' => 'required',
        ]);

        $employee->update($data);

        return response()->json($employee);
    }

    // DELETE /api/employees/{id}
    public function destroy(Employee $employee)
    {
        $employee->delete();

        return response()->json(['message' => 'Karyawan berhasil dihapus.']);
    }
}

Isi app/Http/Controllers/Api/PayrollController.php:

app/Http/Controllers/Api/PayrollController.php 
namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Models\Payroll;
use Illuminate\Http\Request;

class PayrollController extends Controller
{
    // GET /api/payroll — riwayat semua slip gaji
    public function index()
    {
        return Payroll::with('employee')
            ->orderBy('created_at', 'desc')
            ->paginate(10);
    }

    // POST /api/payroll/calculate — hitung THP tanpa simpan
    public function calculate(Request $request)
    {
        $request->validate([
            'basic_salary' => 'required|integer|min:1',
            'allowance'    => 'nullable|integer|min:0',
            'deduction'    => 'nullable|integer|min:0',
        ]);

        $basic     = (int) $request->basic_salary;
        $allowance = (int) ($request->allowance ?? 0);
        $deduction = (int) ($request->deduction ?? 0);
        $net       = ($basic + $allowance) - $deduction;

        return response()->json([
            'basic_salary' => $basic,
            'allowance'    => $allowance,
            'deduction'    => $deduction,
            'net_salary'   => $net,
            'net_formatted' => 'Rp ' . number_format($net, 0, ',', '.'),
        ]);
    }

    // POST /api/payroll — simpan slip gaji ke database
    public function store(Request $request)
    {
        $data = $request->validate([
            'employee_id'  => 'required|exists:employees,id',
            'basic_salary' => 'required|integer|min:1',
            'allowance'    => 'nullable|integer|min:0',
            'deduction'    => 'nullable|integer|min:0',
            'month_year'   => 'required|string',
        ]);

        $data['allowance']  = $data['allowance'] ?? 0;
        $data['deduction']  = $data['deduction'] ?? 0;
        $data['net_salary'] = ($data['basic_salary'] + $data['allowance']) - $data['deduction'];

        $payroll = Payroll::create($data);

        return response()->json(
            $payroll->load('employee'),
            201
        );
    }
}

Langkah 4: Daftarkan Route API

routes/api.php 
use App\Http\Controllers\Api\EmployeeController;
use App\Http\Controllers\Api\PayrollController;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Route;

// Login/register ditangani Breeze (routes/auth.php) — tidak perlu di sini.
// Token untuk testing dibuat via Tinker (lihat Langkah 2b di atas).

// Semua route butuh token Sanctum
Route::middleware('auth:sanctum')->group(function () {

    // Employee API
    Route::apiResource('employees', EmployeeController::class);

    // Payroll API
    Route::get('/payroll', [PayrollController::class, 'index']);
    Route::post('/payroll/calculate', [PayrollController::class, 'calculate']);
    Route::post('/payroll', [PayrollController::class, 'store']);

});

Simpan sebagai Postman Collection

Setelah semua request berhasil, simpan sebagai Collection di Postman (klik Save → masukkan ke collection "Payroll API"). Collection ini bisa di-export dan dibagikan ke teman satu tim, atau dijadikan dokumentasi API proyek.

📋 Rangkuman

  • Tinker cocok untuk test cepat model Eloquent langsung dari terminal — tanpa setup apapun.
  • Postman cocok untuk test HTTP request lengkap persis seperti aplikasi nyata — termasuk headers, auth, body JSON, dan status code response.
  • Sanctum mengamankan API dengan Bearer token. Alur: login → dapat token → pakai token di header semua request berikutnya.
  • Semua logika yang sudah terbukti benar di API (validation, calculation, relationship) akan dipakai ulang oleh Livewire component di Tahap 4 — tidak perlu tulis ulang.
  • API routes di routes/api.php bisa tetap dipertahankan sebagai endpoint untuk integrasi masa depan (misal: app mobile).
