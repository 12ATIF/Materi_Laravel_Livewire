Tahap 2: Autentikasi (Register & Login)

🚀 Tutorial Proyek ⏱️ 55 menit 🔐 Praktik Keamanan

📌 Prasyarat

  • Tampilan Frontend Statis (Tahap 1) telah selesai dibuat.
  • Sudah melakukan koneksi database (konfigurasi file .env).

🎯 Tujuan Belajar

Setelah menyelesaikan materi tahap 2 ini, kamu akan mampu:

  • Memahami struktur Model dan Migrasi bawaan Laravel.
  • Membuat komponen *Livewire* untuk Login secara mandiri agar paham alur datanya secara penuh.
  • Membuat sistem validasi input realtime dan pengamanan *session* pengalihan halaman secara mulus.

🔐 1. Setup Model User & Database

Sistem otentikasi membutuhkan tabel di database. Untungnya, Laravel sudah menyediakannya saat kita melaukan instalasi (*Tabel Users*). Pertama-tama migrasikan skema bawaan database tersebut:

Terminal 
# Jalankan perintah migrasi ini untuk membuat tabel users di database kamu
php artisan migrate

Untuk mencoba login nanti, kita perlu sebuah data dummy (*Seeder*). Buka database/seeders/DatabaseSeeder.php dan aktifkan atau buat satu user admin:

database/seeders/DatabaseSeeder.php 
namespace Database\Seeders;

use Illuminate\Database\Seeder;
use App\Models\User;

class DatabaseSeeder extends Seeder
{
    public function run(): void
    {
        // Buat satu user super admin untuk kita gunakan masuk ke Dashboard
        User::create([
            'name' => 'Admin Payroll',
            'email' => 'admin@payroll.test',
            'password' => bcrypt('password'), // Password otomatis terenkripsi dan hash
        ]);
    }
}

Lalu jalankan php artisan db:seed di terminal.

⚡ 2. Implementasi Livewire Component

Walau Laravel memiliki starter kits sepeerti Breeze / Jetstream, sangat penting kita memahami proses kerjanya dengan merakit logika *Login* milik kita sendiri lewat Component Livewire terbaru. Ini akan memberi kendali penuh.

Terminal 
# Membuat komponen login
php artisan make:livewire auth.login

Logika Backend Login (Controller Livewire)

Buka file *class* yang baru dibuat di direktori root komponen: app/Livewire/Auth/Login.php

app/Livewire/Auth/Login.php 
namespace App\Livewire\Auth;

use Livewire\Component;
use Illuminate\Support\Facades\Auth;

class Login extends Component
{
    // 1. Hubungkan input HTML ke Properties dengan wire:model
    public $email = '';
    public $password = '';

    // 2. Aturan Validasi data yang harus dipenuhi form
    protected $rules = [
        'email' => 'required|email',
        'password' => 'required|min:6',
    ];

    // 3. Method diaktifkan pada tombol Submit
    public function authenticate()
    {
        // Eksekusi Validasi
        $this->validate();

        // Cek apakah email dan password cocok di Database
        if (Auth::attempt(['email' => $this->email, 'password' => $this->password])) {
            // Ciptakan session sukses sehingga sistem tahu user sedang login
            session()->regenerate();
            
            // Lempar pengguna ke halaman Dashboard
            return redirect()->intended('/dashboard');
        }

        // Tampilkan flash error message jika password salah, tanpa me-refresh screen
        session()->flash('error', 'Email atau password salah.');
    }

    public function render()
    {
        // Pastikan kita menggunakan template layout kosong untuk muka login
        return view('livewire.auth.login')->layout('layouts.guest');
    }
}

Menghubungkan Layout Frontend ke Sistem

Ambil template statis Halaman Login yang kita buat di **Tahap 1**, dan ubah agar memiliki fitur ajaib (reaktif) Livewire lewat atribut wire:model dan wire:submit.

resources/views/livewire/auth/login.blade.php 
<div class="bg-gray-100 flex items-center justify-center min-h-screen">
    <div class="bg-white p-8 rounded-lg shadow-md w-full max-w-md">
        <div class="mb-6 text-center">
            <h2 class="text-2xl font-bold text-gray-800">Payroll System</h2>
        </div>

        <!-- Flash Message Jika Auth Gagal -->
        @if (session()->has('error'))
            <div class="bg-red-100 border border-red-400 text-red-700 px-4 py-3 rounded relative mb-4">
                <span class="block sm:inline">{{ session('error') }}</span>
            </div>
        @endif

        <!-- Hubungkan penekanan Tombol Submit dengan method "authenticate" di PHP -->
        <form wire:submit="authenticate">
            <div class="mb-4">
                <label class="block text-sm font-medium text-gray-700">Email</label>
                <!-- PENTING: wire:model.blur akan memvalidasi data sesaat kursor pengguna pindah (hilang fokus) dari kolom ini -->
                <input type="email" wire:model.blur="email" class="mt-1 block w-full rounded-md border-gray-300 shadow-sm focus:border-blue-500" placeholder="admin@payroll.test">
                <!-- Tampilan Error Validasi (Realtime) -->
                @error('email') <span class="text-sm text-red-600">{{ $message }}</span> @enderror
            </div>

            <div class="mb-6">
                <label class="block text-sm font-medium text-gray-700">Password</label>
                <input type="password" wire:model.blur="password" class="mt-1 block w-full rounded-md border-gray-300 shadow-sm focus:border-blue-500" placeholder="••••••••">
                @error('password') <span class="text-sm text-red-600">{{ $message }}</span> @enderror
            </div>

            <button type="submit" class="w-full bg-blue-600 text-white font-bold py-2 px-4 rounded hover:bg-blue-700">
                Masuk
            </button>
        </form>
    </div>
</div>

📚 Alur Proses Kerja Autentikasi Ini:

  1. Form Submit: Pengguna mengklik "Masuk". Form memicu wire:submit="authenticate" yang akan melempar nilai ke kelas PHP.
  2. Validasi Data: Nilai $email dan $password diolah oleh $this->validate(). Jika pengguna mengisi form asal-asalan, maka sistem tidak melanjutkannya, melempar kembali Error Validation ke dalam @error('field') di HTML secara kilat tanpa layar memuat (loading).
  3. Auth Facade (Session): Fungsi khusus Auth::attempt() bawaan Laravel secara rahasia akan memeriksa database. Jika informasi sama, maka ia akan membuat sidik jari sementara (session file / cookie) di browser yang memberitahu seluruh aplikasi bahwa "user ini aman, beri dia jalan masuk".
  4. Redirect: Karena sudah diyakinkan keamanannya, Livewire akan merekonstruksi ulang (regenerate()) memori server untuk menjaga dari peretasan sesi dan otomatis melontarkan si pengguna (*redirect*) ke /dashboard.

📋 Ringkasan Tahap 2

  • Dalam implementasi sistem Login terintegrasi Livewire 3 terbaru, tidak diperlukan pembuatan Routing `POST` di dalam web.php. Karena sistem reaktif komponen Livewire merangkap tanggung jawab tersebut.
  • Kami menggunakan wire:model.blur. Akhiran `.blur` berarti validasi di kelas PHP baru ditembak saat si user sudah mengetik tuntas dan mengklik tempat lain, meminimalisir lag internet.
  • Fungsi rahasia Laravel: Auth::attempt() bertanggung jawab dari urusan pencocokan string biasa (email) dan pencocokan Hash acak bcrypt yang sukar ditebak untuk (password). Semua sudah ditangani di balik layar!
