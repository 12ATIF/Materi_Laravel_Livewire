Tahap 2: Autentikasi (Register & Login)

🚀 Tutorial Proyek ⏱️ 55 menit 🔐 Praktik Keamanan
⚙️ Versi: 🔥 Laravel 13.x ⚡ Livewire 3.x 🐘 PHP 8.3+ 🛡️ Laravel Breeze Kita pakai Laravel Breeze stack Livewire — starter kit resmi Laravel yang menghasilkan auth (login, register, forgot password) secara otomatis berbasis Livewire. Tidak perlu rakit form login dari nol.

📌 Prasyarat

  • Tampilan Frontend Statis (Tahap 1) telah selesai dibuat.
  • Database sudah dibuat dan .env sudah dikonfigurasi (DB_DATABASE, DB_USERNAME, DB_PASSWORD).
  • Node.js sudah terinstall (cek dengan node -v di terminal).

🎯 Tujuan Belajar

Setelah menyelesaikan materi tahap 2 ini, kamu akan mampu:

  • Menginstall dan mengkonfigurasi Laravel Breeze dengan Livewire stack.
  • Memahami file-file yang dihasilkan Breeze: layout, komponen auth, dan route.
  • Mengkustomisasi layout app.blade.php untuk navigasi aplikasi payroll.
  • Melindungi halaman dengan middleware auth agar hanya user yang login yang bisa akses.
  • Membuat data admin pertama lewat Seeder.

🛡️ Apa itu Laravel Breeze?

Laravel Breeze adalah starter kit autentikasi resmi dari Laravel. Dengan satu perintah, Breeze langsung menghasilkan halaman login, register, lupa password, reset password, dan verifikasi email — semuanya sudah jadi dan bisa langsung dipakai.

💡

Kenapa pakai Breeze dan bukan buat sendiri?

Autentikasi adalah fitur yang kritis dari sisi keamanan. Breeze sudah mengikuti praktik terbaik Laravel: password di-hash dengan bcrypt, session diregenerasi saat login, CSRF protection aktif otomatis. Membuatnya sendiri dari nol berisiko terlewat satu celah keamanan. Di dunia kerja pun, developer profesional pakai Breeze atau Jetstream — bukan buat dari nol.

📦 1. Instalasi Laravel Breeze

Jalankan 4 perintah ini secara berurutan di terminal, di dalam folder project Laravel kamu:

Terminal — urutan wajib diikuti 
# 1. Download package Breeze
composer require laravel/breeze --dev

# 2. Generate semua file auth (pilih stack: livewire)
php artisan breeze:install livewire

# 3. Install dependency JavaScript (Tailwind, Vite, dll)
npm install

# 4. Buat tabel users dan semua tabel bawaan Laravel
php artisan migrate
📌

Saat breeze:install ditanya pilihan — jawab tepat ini:

  • Stack: pilih Livewire
  • Livewire stack: pilih functional (Volt) — BUKAN "class". Pilihan ini menentukan struktur file auth yang dihasilkan!
  • Dark mode: pilih No (agar lebih sederhana)
  • Testing: pilih PHPUnit (default)
⚠️

Kenapa harus pilih functional (Volt)?

Pilihan "functional" menghasilkan halaman auth sebagai file .blade.php tunggal (Volt syntax). Pilihan "class" menghasilkan file PHP terpisah di app/Livewire/. Tutorial ini mengajarkan struktur functional (Volt) — jika kamu salah pilih "class", struktur foldernya akan berbeda dan langkah berikutnya tidak akan cocok.

📁 2. File yang Dihasilkan Breeze

Setelah breeze:install, Breeze menghasilkan banyak file secara otomatis. Ini yang paling penting untuk dipahami:

💡

Breeze Livewire Pakai Teknologi Volt

Berbeda dari Breeze biasa yang membuat file PHP terpisah untuk setiap halaman auth, Breeze stack Livewire memakai Livewire Volt — logika PHP dan tampilan Blade digabung dalam satu file .blade.php. Itulah kenapa kamu tidak akan menemukan folder app/Livewire/Auth/.

File / Folder Fungsi
resources/views/layouts/app.blade.php Kerangka HTML utama untuk halaman yang butuh login — berisi <livewire:layout.navigation /> dan {{ $slot }}
resources/views/layouts/guest.blade.php Kerangka HTML untuk halaman tanpa login (login, register)
resources/views/livewire/layout/navigation.blade.php Komponen navbar — inilah tempat kita menambahkan menu Karyawan & Payroll
resources/views/livewire/pages/auth/*.blade.php Halaman login, register, forgot-password, dst — pakai Volt (logika + tampilan dalam 1 file)
app/Livewire/Forms/LoginForm.php Class form khusus login dengan validasi dan rate-limiting
app/Livewire/Actions/Logout.php Action class untuk proses logout (invalidate session)
routes/auth.php Semua route auth menggunakan sintaks Volt::route() — bukan Route::get() biasa
resources/views/dashboard.blade.php Halaman dashboard default setelah login
⚠️

Jangan Edit File Auth Breeze!

File di resources/views/livewire/pages/auth/ sudah aman dan teruji. Kita tidak perlu menyentuhnya. Yang akan kita kustomisasi hanya resources/views/livewire/layout/navigation.blade.php untuk menambahkan menu payroll ke navbar.

🎨 3. Menambah Menu ke Navbar Payroll

Navbar di Breeze Livewire bukan ada di layouts/app.blade.php — file itu hanya berisi kerangka dan memanggil <livewire:layout.navigation />. Navigasi sebenarnya ada di file terpisah.

Buka resources/views/livewire/layout/navigation.blade.php — cari bagian yang berisi route('dashboard') (link Dashboard bawaan), lalu tambahkan 3 link baru tepat di bawahnya:

resources/views/livewire/layout/navigation.blade.php — bagian nav links 
<!-- Sudah ada bawaan Breeze (jangan dihapus): -->
<x-nav-link :href="route('dashboard')" :active="request()->routeIs('dashboard')" wire:navigate>
    {{ __('Dashboard') }}
</x-nav-link>

<!-- Tambahkan 3 baris ini tepat setelahnya: -->
<x-nav-link :href="route('employees.index')" :active="request()->routeIs('employees.*')" wire:navigate>
    Karyawan
</x-nav-link>

<x-nav-link :href="route('payroll.calculator')" :active="request()->routeIs('payroll.calculator')" wire:navigate>
    Payroll
</x-nav-link>

<x-nav-link :href="route('payroll.history')" :active="request()->routeIs('payroll.history')" wire:navigate>
    Riwayat Gaji
</x-nav-link>
⚠️

Jangan Lupa wire:navigate!

Semua <x-nav-link> bawaan Breeze sudah pakai atribut wire:navigate. Pastikan link yang kamu tambahkan juga pakai atribut ini — tanpanya navigasi antar halaman akan full page reload dan kehilangan keunggulan SPA Livewire.

🔍

Ada 2 Blok Nav yang Perlu Diubah

Di dalam navigation.blade.php, ada 2 blok x-nav-link yang serupa: satu untuk tampilan desktop dan satu untuk tampilan mobile (biasanya ditandai class responsive atau berada di dalam x-responsive-nav-link). Tambahkan menu payroll di kedua blok agar tampil di HP maupun laptop.

🌱 4. Membuat Data Admin (Seeder)

Agar bisa langsung login dan testing tanpa harus Register terlebih dahulu, kita buat akun admin via Seeder.

database/seeders/DatabaseSeeder.php 
namespace Database\Seeders;

use Illuminate\Database\Seeder;
use App\Models\User;
use Illuminate\Support\Facades\Hash;

class DatabaseSeeder extends Seeder
{
    public function run(): void
    {
        // firstOrCreate: aman dijalankan berkali-kali, tidak error jika email sudah ada
        User::firstOrCreate(
            ['email' => 'admin@payroll.test'],
            [
                'name'              => 'Admin Payroll',
                'password'          => Hash::make('password'),
                'email_verified_at' => now(), // ← wajib! tanpa ini admin tidak bisa login jika verifikasi email aktif
            ]
        );
    }
}
Terminal 
php artisan db:seed

🔒 5. Melindungi Route dengan Middleware auth

Setelah semua halaman karyawan dan payroll dibuat di tahap selanjutnya, kita akan melindunginya agar hanya bisa diakses oleh user yang sudah login. Caranya dengan menambahkan ->middleware('auth') di route.

routes/web.php — contoh proteksi route 
// ✅ Route ini HANYA bisa diakses jika sudah login
Route::middleware('auth')->group(function () {
    Route::get('/karyawan', EmployeeManager::class)->name('employees.index');
    Route::get('/payroll', PayrollCalculator::class)->name('payroll.calculator');
    Route::get('/riwayat-gaji', PayrollHistory::class)->name('payroll.history');
});

// ❌ Jika belum login dan akses /karyawan → otomatis diarahkan ke /login

Route lengkap project ini akan ditampilkan di Tahap 5 setelah semua komponen selesai dibuat.

✅ 6. Verifikasi: Coba Login

Setelah semua langkah di atas, jalankan development server:

Terminal — buka 2 tab terminal sekaligus 
# Tab 1: jalankan Laravel
php artisan serve

# Tab 2: compile Tailwind CSS (harus tetap berjalan saat development)
npm run dev

Buka browser, akses http://localhost:8000/login dan masuk dengan:

  • Email: admin@payroll.test
  • Password: password
🎯

Halaman yang Sudah Aktif Setelah Breeze:

  • /login — form login Livewire
  • /register — form register Livewire
  • /dashboard — halaman setelah login (butuh auth)
  • /forgot-password — kirim link reset password
  • /profile — edit profil user

Semua ini sudah jadi tanpa menulis satu baris kode auth pun!

📚 Cara Kerja Auth Breeze di Balik Layar

  1. User submit form login → Volt component resources/views/livewire/pages/auth/login.blade.php menangkap data via wire:submit dan mendelegasikan ke app/Livewire/Forms/LoginForm.php.
  2. Validasi + Rate Limit → LoginForm memvalidasi email/password dan menerapkan throttle 5 percobaan per IP untuk mencegah brute force.
  3. Auth::attempt() → Laravel mencocokkan email ke database dan membandingkan password yang diinput dengan hash bcrypt yang tersimpan. Jika cocok, session dibuat.
  4. Redirect → User diarahkan ke /dashboard via $this->redirectIntended() — atau ke halaman yang dicoba diakses sebelumnya.
  5. Middleware auth + verified → Dashboard dilindungi dua middleware: auth (cek login) dan verified (cek email sudah dikonfirmasi). Itulah kenapa kita perlu set email_verified_at di Seeder.

📋 Ringkasan Tahap 2

  • Breeze = 4 perintah: composer require → breeze:install livewire → npm install → migrate. Setelah itu auth langsung bisa dipakai.
  • Breeze Livewire memakai Volt — halaman auth adalah file .blade.php yang berisi logika PHP sekaligus. Tidak ada folder app/Livewire/Auth/.
  • Navbar ada di resources/views/livewire/layout/navigation.blade.php — bukan di layouts/app.blade.php. Tambahkan menu payroll di sana, pakai wire:navigate.
  • Dashboard dilindungi dua middleware: auth dan verified. Karena itu Seeder harus set email_verified_at => now().
  • Jalankan 2 terminal sekaligus saat development: php artisan serve dan npm run dev.
