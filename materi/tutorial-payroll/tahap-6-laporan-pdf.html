Tahap 6: Ekspor Data ke PDF & Report

🚀 Tutorial Proyek (Bonus Lanjutan) ⏱️ 40 menit 🖨️ Praktik Integrasi Package
⚙️ Versi: 🔥 Laravel 13.x ⚡ Livewire 3.x 🐘 PHP 8.3+ 📄 barryvdh/laravel-dompdf ^3.0 Pakai package barryvdh/laravel-dompdf untuk export PDF. Pastikan ekstensi PHP gd dan mbstring aktif di Laragon.

🎯 Tujuan Belajar

Bagian ini adalah persembahan khusus untuk merubah aplikasi Anda menjadi standar Dunia Industri:

  • Menggunakan Composer untuk mengunduh package alat pihak ketiga (DOMPDF Laravel).
  • Memahami kelemahan desain CSS rumit milik Tailwind yang tidak kompatibel penuh dengan PDF Engine tipe lama (HTML ke PDF).
  • Menciptakan rute (*route*) murni laravel untuk mengunduh (*download*) berkas bukti otentik slip gaji bulanan karyawan.

🔧 0. Aktifkan Extension PHP yang Diperlukan (Laragon)

DOMPDF membutuhkan extension PHP gd dan mbstring. Di Laragon, keduanya biasanya sudah aktif, tapi jika ada error saat generate PDF, ikuti langkah berikut:

⚙️

Cara Aktifkan Extension di Laragon:

  1. Klik kanan ikon Laragon di taskbar → PHP → php.ini
  2. Cari baris ;extension=gd → hapus titik koma di depannya → jadi extension=gd
  3. Cari baris ;extension=mbstring → hapus titik koma → jadi extension=mbstring
  4. Simpan file, lalu restart Laragon (klik kanan → Reload)

📦 1. Instalasi Library DOMPDF

Laravel tidak punya fitur export PDF bawaan. Kita pakai package populer barryvdh/laravel-dompdf:

Terminal 
# Unduh package via Composer
composer require barryvdh/laravel-dompdf
✅

Komponen PayrollHistory sudah dibuat di Tahap 5!

Kamu tidak perlu membuat ulang. Di Tahap 5, kita sudah membuat komponen app/Livewire/PayrollHistory.php beserta view payroll-history.blade.php dengan fitur filter periode dan pagination.

Yang perlu dilakukan di Tahap 6 ini hanya 2 hal:

  1. Install DOMPDF (sudah dilakukan di Section 1 atas)
  2. Buat file resources/views/pdf/slip-gaji.blade.php + tambahkan route cetak ke routes/web.php

Setelah route payroll.cetak terdefinisi, tombol "🖨️ Cetak PDF" di halaman Riwayat Gaji akan otomatis muncul!

🖨️ 2. Mendesain Layout Kertas PDF

⚠️ Perhatian! PDF tidak suka Tailwind Modern!

Engine `dompdf` adalah konverter berbasis HTML4 dan CSS lama konvensional. Ia AKAN BERANTAKAN jika dipaksa menelan class buatan modern seperti `flex`, `grid`, dsb dari Tailwind. Oleh sebab itu kita harus menggunakan CSS Table Kuno (Vanilla HTML & Inline CSS) untuk menjamin presisi posisi garis dan teks di dalam PDF.

🔗 3. Daftarkan Route Cetak PDF ke routes/web.php

⚠️

Ini TAMBAHAN ke web.php yang sudah ada dari Tahap 5 — bukan file baru!

Buka routes/web.php yang sudah kita isi di Tahap 5. Tambahkan 2 baris use di paling atas file, dan 1 route cetak di dalam Route::middleware('auth')->group() yang sudah ada. Jangan timpa seluruh file!

routes/web.php — tambahkan use + 1 route di dalam group auth 
// Tambahkan 2 baris ini di bagian atas file (setelah use yang sudah ada)
use App\Models\Payroll;
use Barryvdh\DomPDF\Facade\Pdf;

// ... (semua route dari Tahap 5 tetap ada) ...

// Tambahkan route ini DI DALAM Route::middleware('auth')->group(function () { ...
Route::get('/cetak-slip/{id}', function ($id) {
    $payroll = Payroll::with('employee')->findOrFail($id);
    $pdf = Pdf::loadView('pdf.slip-gaji', ['data' => $payroll]);
    return $pdf->stream('Slip_Gaji_' . $payroll->employee->nik . '.pdf');
})->name('payroll.cetak'); // ← name() WAJIB ada! blade memanggil route('payroll.cetak', id)
🔗

Kenapa ->name('payroll.cetak') tidak boleh dihilangkan?

File payroll-history.blade.php (Tahap 5) memanggil route('payroll.cetak', $p->id) untuk membangun URL tombol Cetak. Tanpa nama route ini, Laravel akan throw RouteNotFoundException dan seluruh halaman Riwayat Gaji crash.
