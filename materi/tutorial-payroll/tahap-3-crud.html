Tahap 3: CRUD Master Data Karyawan

🚀 Tutorial Proyek ⏱️ 60 menit ⚙️ Database & Livewire
⚙️ Versi: 🔥 Laravel 13.x ⚡ Livewire 3.x 🐘 PHP 8.3+ 🗄️ MySQL / MariaDB Migration default Laravel 11 sudah pakai struktur baru (tanpa Kernel.php). Komponen Livewire ada di app/Livewire/.

📌 Prasyarat

  • Pemahaman dasar tentang Model dan Migrasi Database di Laravel.
  • Sudah menguasai materi penggunaan dasar wire:click dan wire:model dari Minggu ke-2.

🎯 Tujuan Belajar

Setelah menyelesaikan materi tahap 3 ini, kamu akan mampu:

  • Mengimplementasikan konsep Create, Read, Update, Delete (CRUD) utuh dalam satu komponen Livewire saja (Single Page Application).
  • Menghubungkan rancangan Mockup HTML menjadi komponen yang benar-benar bisa menympan data ke MySQL.
  • Memahami sifat "Reaktif" (State) pada Public Property di dalam Livewire Controller.

📂 1. Membuat Model & Database Migration

Sistem aplikasi penggajian tidak akan berjalan tanpa adanya entitas Karyawan (Employee). Mari kita siapkan Model dan *tabel* databasenya terlebih dahulu.

Terminal 
# Membuat Model sekaligus Migration (-m)
php artisan make:model Employee -m

Buka file migrasi yang barusan terbuat di database/migrations/xxxx_xx_xx_xxxxxx_create_employees_table.php dan lengkapi kolom-kolomnya:

*_create_employees_table.php 
public function up(): void
{
    Schema::create('employees', function (Blueprint $table) {
        $table->id();
        $table->string('nik')->unique();      // Nomor Induk Karyawan
        $table->string('name');             // Nama Lengkap
        $table->string('position');         // Jabatan (HRD/Staff/Keuangan dll)
        $table->timestamps();
    });
}

Tambahkan Fillable pada Model. Buka app/Models/Employee.php:

app/Models/Employee.php 
class Employee extends Model
{
    protected $fillable = ['nik', 'name', 'position'];

    // Satu karyawan bisa punya banyak slip gaji (relasi ke Tahap 4)
    public function payrolls()
    {
        return $this->hasMany(\App\Models\Payroll::class);
    }
}
🔗

Relasi hasMany di Employee — Untuk Apa?

Baris $this->hasMany(Payroll::class) mendefinisikan bahwa satu karyawan bisa punya banyak slip gaji. Model Payroll baru akan dibuat di Tahap 4 — jadi baris ini masih akan error jika dicoba sekarang, dan itu normal. Relasi ini akan aktif dan berguna setelah Tahap 4 selesai, misalnya: $employee->payrolls langsung menampilkan semua riwayat gajinya.

Jangan lupa eksekusi perubahan database dengan mengetikkan perintah `php artisan migrate` di terminal.

⚡ 2. Membuat Komponen Livewire Manajemen Karyawan

Kita akan membuat otak yang membawahi form tambah karyawan beserta tabel data yang tadi kita buat di *Tahap 1* ke dalam satu komponen sakti: EmployeeManager.

Terminal 
# Membuat komponen Livewire Karyawan
php artisan make:livewire EmployeeManager

A. Mendefinisikan Logika Backend Karyawan (Class Livewire)

Buka app/Livewire/EmployeeManager.php. Di sini kita membuat fungsi Tambah, Edit, Hapus (CRUD).

app/Livewire/EmployeeManager.php 
namespace App\Livewire;

use Livewire\Component;
use App\Models\Employee;

class EmployeeManager extends Component
{
    public ?int $employee_id = null;
    public bool $isEditMode = false;

    public string $name = '';
    public string $nik = '';
    public string $position = '';

    // 1. CREATE / UPDATE
    public function store()
    {
        // Satu titik validasi: NIK unique dikecualikan untuk data yang sedang diedit
        $this->validate([
            'nik'      => 'required|unique:employees,nik,' . ($this->employee_id ?? 'NULL'),
            'name'     => 'required|min:3',
            'position' => 'required',
        ], [
            'nik.required'      => 'NIK wajib diisi.',
            'nik.unique'        => 'NIK ini sudah digunakan karyawan lain.',
            'name.required'     => 'Nama karyawan wajib diisi.',
            'name.min'          => 'Nama minimal 3 karakter.',
            'position.required' => 'Jabatan wajib dipilih.',
        ]);

        Employee::updateOrCreate(
            ['id' => $this->employee_id],
            [
                'nik'      => $this->nik,
                'name'     => $this->name,
                'position' => $this->position,
            ]
        );

        session()->flash('success', $this->isEditMode ? 'Data karyawan berhasil diperbarui.' : 'Karyawan baru berhasil ditambahkan.');
        $this->resetForm();
    }

    // 2. Siapkan form Edit
    public function edit(int $id)
    {
        $emp = Employee::findOrFail($id);
        $this->employee_id = $emp->id;
        $this->nik         = $emp->nik;
        $this->name        = $emp->name;
        $this->position    = $emp->position;
        $this->isEditMode  = true;
    }

    // 3. DELETE
    public function delete(int $id)
    {
        Employee::findOrFail($id)->delete();
        session()->flash('success', 'Karyawan berhasil dihapus.');
    }

    public function resetForm()
    {
        $this->reset(['employee_id', 'nik', 'name', 'position', 'isEditMode']);
        $this->resetValidation();
    }

    // 4. READ
    public function render()
    {
        return view('livewire.employee-manager', [
            'employees' => Employee::orderBy('id', 'desc')->get(),
        ])->layout('layouts.app');
    }
}

B. Menghubungkan Backend ke Frontend Mockup HTML (Blade View)

Ambil template statis `employee` dari **Tahap 1**. Timpa (Replace) isinya ke dalam file resources/views/livewire/employee-manager.blade.php, dan bubuhkan perintah-perintah sakti Livewire seperti wire:click, wire:model, dan logika Looping Blade (@foreach)!

resources/views/livewire/employee-manager.blade.php 
<!-- Bungkus komponen dalam satu root element -->
<div>

<!-- ✅ Flash Message Notifikasi Sukses -->
@if (session()->has('success'))
    <div class="max-w-7xl mx-auto px-4 pt-4">
        <div class="bg-green-100 border border-green-400 text-green-800 px-4 py-3 rounded">
            ✅ {{ session('success') }}
        </div>
    </div>
@endif

<div class="max-w-7xl mx-auto px-4 grid grid-cols-1 md:grid-cols-3 gap-8 py-8">

    <!-- KOLOM KIRI: Form Tambah Karyawan -->
    <div class="bg-white p-6 rounded-lg shadow border border-gray-100 md:col-span-1 h-fit">
        <!-- Judul Form Berubah Sesuai State (Reaktif) -->
        <h3 class="text-lg font-bold text-gray-800 mb-4">
            {{ $isEditMode ? 'Edit Karyawan' : 'Input Karyawan Baru' }}
        </h3>

        <!-- Submit form dikaitkan dengan fungsi "store" -->
        <form wire:submit="store">
            <div class="mb-4">
                <label class="block text-sm font-medium text-gray-700">Nama Lengkap</label>
                <input type="text" wire:model.blur="name" class="mt-1 block w-full rounded border-gray-300 shadow-sm">
                @error('name') <span class="text-xs text-red-500">{{ $message }}</span> @enderror
            </div>
            <div class="mb-4">
                <label class="block text-sm font-medium text-gray-700">Nomor Induk / NIK</label>
                <input type="text" wire:model.blur="nik" class="mt-1 block w-full rounded border-gray-300 shadow-sm">
                @error('nik') <span class="text-xs text-red-500">{{ $message }}</span> @enderror
            </div>
            <div class="mb-4">
                <label class="block text-sm font-medium text-gray-700">Jabatan</label>
                <select wire:model="position" class="mt-1 block w-full rounded border-gray-300 shadow-sm">
                    <option value="">-- Pilih Jabatan --</option>
                    <option value="Staff IT">Staff IT</option>
                    <option value="HRD / Personalia">HRD / Personalia</option>
                    <option value="Keuangan">Keuangan</option>
                </select>
                @error('position') <span class="text-xs text-red-500">{{ $message }}</span> @enderror
            </div>

            <div class="flex gap-2">
                <button type="submit"
                    wire:loading.attr="disabled"
                    class="flex-1 bg-blue-600 text-white font-bold py-2 rounded shadow disabled:opacity-50">
                    <span wire:loading.remove>Simpan</span>
                    <span wire:loading>Menyimpan...</span>
                </button>
                
                <!-- Tampilkan tombol Batal HANYA saat Mode Edit menyala -->
                @if($isEditMode)
                    <button type="button" wire:click="resetForm" class="flex-1 bg-gray-500 text-white font-bold py-2 rounded shadow">Batal</button>
                @endif
            </div>
        </form>
    </div>

    <!-- KOLOM KANAN: Tabel Karyawan -->
    <div class="bg-white p-6 rounded-lg shadow border border-gray-100 md:col-span-2 overflow-x-auto">
        <h3 class="text-lg font-bold text-gray-800 mb-4">Daftar Karyawan</h3>
        <table class="min-w-full divide-y divide-gray-200 text-sm">
            <thead>
                <tr class="bg-gray-50 text-left">
                    <th class="px-4 py-2 text-gray-500">NIK</th>
                    <th class="px-4 py-2 text-gray-500">Nama</th>
                    <th class="px-4 py-2 text-gray-500">Jabatan</th>
                    <th class="px-4 py-2 text-gray-500">Aksi</th>
                </tr>
            </thead>
            <tbody class="divide-y divide-gray-200">
                
                <!-- Kita lakukan iterasi Database (Looping) -->
                @forelse($employees as $item)
                    <tr>
                        <td class="px-4 py-3">{{ $item->nik }}</td>
                        <td class="px-4 py-3 font-semibold">{{ $item->name }}</td>
                        <td class="px-4 py-3">{{ $item->position }}</td>
                        <td class="px-4 py-3">
                            <!-- Memanggil fungsi public yg ada di komponen PHP, disertai Parameter lemparan ID -->
                            <button wire:click="edit({{ $item->id }})" class="text-indigo-600 hover:underline mr-2">Edit</button>
                            
                            <!-- Menggunakan bawaan Javascript confirm Dialog untuk keamanan sebelum Delete -->
                            <button wire:click="delete({{ $item->id }})" wire:confirm="Yakin ingin menghapus karyawan ini?" class="text-red-600 hover:underline">Hapus</button>
                        </td>
                    </tr>
                @empty
                    <tr>
                        <td colspan="4" class="px-4 py-6 text-center text-gray-500">Data Karyawan masih kosong.</td>
                    </tr>
                @endforelse

            </tbody>
        </table>
    </div>
</div>
</div> <!-- penutup div root terluar -->

📚 Analisa Mekanisme Penting Livewire:

  • Reactive Properties ($isEditMode): Perhatikan bagaimana teks Judul Form berubah otomatis ("Input Karyawan Baru" vs "Edit Karyawan") dan memunculkan tombol Batal jika tombol edit diklik tanpa layar berkedip (`reload`). Hal ini berkat pengkondisian {{ $isEditMode ? ... }} di file Blade yang selalu dipantau *state* perubahannya oleh sistem Livewire.
  • wire:model vs wire:model.blur: Anda telah mengkoneksikan kotak input *Nama*, *NIK* dengan isi memori controller ($nik). Ketika Anda mengetik di sana, isinya diterjemahkan dan menyatu dengan *Controller* di *Backend* secara halus. Karena kita menggunakan format akhiran `.blur` untuk NIK/Nama, ini berarti inputan akan dikirim ke Backend pada saat kursor terlepas dari *input box* tersebut (untuk menghemat jaringaan internet dibandingkan `wire:model.live` konstan).
  • Parameter `wire:click`: Pada tombol edit dan hapus (wire:click="edit({{ $item->id }})"), kamu mengirim ID dari database spesifik secara parameter balik ke kelas komponen, membukitkan kemudahan sistem Data Passing dari PHP-Blade-PHP.
  • Konfirmasi Keamanan (wire:confirm): Livewire versi 3 menyematkan fitur bawaan pemanggilan Javascript `window.confirm`. Jadi apabila tombol Hapus tak sengaja terpencet, user ditanya dulu konfirmasi konvensional sebelum delete() PHP tereksekusi.

🔗 Daftarkan Route Halaman Karyawan

Setelah komponen selesai dibuat, daftarkan route-nya di routes/web.php agar bisa diakses dari browser:

routes/web.php 
use App\Livewire\EmployeeManager;

Route::middleware('auth')->group(function () {
    Route::get('/karyawan', EmployeeManager::class)->name('employees.index');
    // ... route lain akan ditambahkan di tahap berikutnya
});

Akses halaman ini di browser: http://localhost:8000/karyawan (pastikan sudah login).

📋 Ringkasan Tahap 3

  • Anda sekarang punya aplikasi yang memiliki siklus hidup CRUD (Daftar, Baca, Ubah, Hapus) Karyawan Lengkap dan Berfungsi Nyata.
  • Kita telah melihat kedahsyatan form `Single Page Application`. Form Input dan Tabel ada dalam wadah yang sama. Saat berhasil ditekan "Simpan", Tabel bergeser ter-*Refresh*, form ter-*Reset* merespons state terbaru—bagaikan menyusutkan belasan baris kode *jQuery Ajax* dan beberapa Controller standard menjadi hanya *SATU Class Livewire Component Saja*.
