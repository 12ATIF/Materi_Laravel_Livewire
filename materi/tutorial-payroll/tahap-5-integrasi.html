Tahap 5: Menerapkan Backend ke Frontend

🚀 Tutorial Proyek ⏱️ 40 menit 🔮 Praktik Perbandingan Code

🎯 Tujuan Belajar

Welcome to the Finale! Setelah melalui seluruh fase pembuatan arsitektur di tahap sebelumnya, mari kita bedah *What exactly happened* (Apa yang sesungguhnya terjadi). Pada materi ini kita akan:

  • Mengkaji ulang (*Review*) perbandingan drastis antara kode HTML statis (Tahap 1) dengan kode Blade + Livewire dinamis terbaru (Tahap 2-4).
  • Memahami kedahsyatan `wire:model` dalam menyambung input form dengan PHP secara tanpa kelim (*seamless*).
  • Mengganti metrik dummy di Dashboard menjadi nilai Query Eloquent asli.

🔍 1. Anatomi Perubahan: Form Payroll (Sebelum vs Sesudah)

Mari sandingkan perubahan rancangan UI Kalkulator Payroll yang awalnya "mati" tanpa nyawa di Tahap 1 melawan versi "hidup"-nya yang kita desain di Tahap 4.

❌ Tahap 1 (Statis - HTML Murni biasa) 
<!-- Dropdown Karyawan Mati (Hardcoded) -->
<select class="w-full rounded-md border-gray-300">
    <option disabled selected>-- Cari Karyawan --</option>
    <option>EMP-001 | Budi Santoso</option>
</select>

<!-- Input Nominal Mati -->
<input type="number" class="w-full rounded border-gray-300" placeholder="Rp 0">

<!-- Take Home Pay Hasil Angka Mati -->
<div class="text-3xl font-extrabold text-blue-600">
    Rp 0
</div>
✅ TAHAP 4 (Dinamis - Livewire Component Blade) 
<!-- Dropdown Dinamis mengambil referensi dari properti $employees dari DB -->
<select wire:model="employee_id" class="w-full rounded-md border-gray-300">
    <option value="">-- Cari Karyawan --</option>
    @foreach($employees as $emp)
        <option value="{{ $emp->id }}">{{ $emp->nik }} | {{ $emp->name }}</option>
    @endforeach
</select>

<!-- Input hidup diikat ke Backend, menembakkan Hook updated() saat diubah -->
<input wire:model.live="basic_salary" type="number" class="w-full rounded border-gray-300">

<!-- Take Home Pay tercetak otomatis merubah digit merespons state property PHP -->
<div class="text-3xl font-extrabold text-blue-600">
    Rp {{ number_format($net_salary, 0, ',', '.') }}
</div>

💡 Penjelasan: Apa fungsi `wire:model`?

Singkatnya: Jembatan Dua Arah (Two-Way Binding).

Ketika kamu memberikan atribut wire:model.live="basic_salary" pada input teks HTML, nilai yang diketik oleh pengguna akan langsung menyebrang jaringan internet menuju Variabel Public di PHP (public $basic_salary). Begitu juga sebaliknya, jika kamu memerintahkan PHP mengubah nilainya, angka di layar browser pengguna otomatis tergantikan tanpa kedipan layar (Page Reload).

📊 2. Finalisasi Metrik Halaman Dashboard

Pada file statis awal (static/dashboard.blade.php) kita mengisikan angka palsu (seperti "24 Orang" Karyawan). Kini mari kita terapkan perhitungan nyata. Kamu disarankan untuk merubah Dashboard bawaan Breeze lalu melemparkan kalkulasi data nyata lewat komponen Livewire.

Mengekstrak data Realtime di Dashboard Component Anda 
// Query PHP yang disisipkan:
public function render() {
    return view('livewire.dashboard', [
        'total_employee' => \App\Models\Employee::count(), 
        'total_payroll_month' => \App\Models\Payroll::whereMonth('created_at', date('m'))->sum('net_salary')
    ]);
}
Merubah tag HTML di resources/views/livewire/dashboard.blade.php 
<!-- Angka Palsu Terdahulu -->
<p class="text-3xl font-bold text-gray-800">24 Orang</p>

<!-- Menjadi Dinamis (Terkoneksi Backend) -->
<p class="text-3xl font-bold text-gray-800">{{ $total_employee }} Orang</p>

<!-- Menampilkan Agregat Keuangan Bulan Ini -->
<p class="text-3xl font-bold text-gray-800">Rp {{ number_format($total_payroll_month, 0, ',', '.') }}</p>

🎉 KESIMPULAN TUTORIAL PAYROLL

  • Selamat! Anda telah me-refactoring desain sistem UI statis menjadi Aplikasi Penggajian dengan tumpukan teknologi modern: TALL Stack (Tailwind, Alpine.JS, Laravel, Livewire).
  • Sistem Data Binding Livewire merupakakan metode tercanggih untuk para dominasi pengembang PHP dalam bersaing dengan teknologi Front-End berat seperti React JS yang membutuhkan instalasi berbelit-belit.
  • Aplikasi ini adalah tapak mula (steopping stone). Pada dunia kerja kalian dapat memolesnya lebih jauh, menyisipkan Login Role/Multi-auth, Modul Cetak PDF (*menggunakan facade DOMPDF*), dan masih banyak lagi kemungkinan lainnya!
