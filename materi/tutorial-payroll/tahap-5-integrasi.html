Tahap 5: Integrasi Akhir

🚀 Tutorial Proyek ⏱️ 40 menit 🔮 Praktik Perbandingan Code
⚙️ Versi: 🔥 Laravel 13.x ⚡ Livewire 3.x 🐘 PHP 8.3+ Tahap ini menggabungkan view Tailwind dengan logika Livewire. Pastikan Vite dev server berjalan (npm run dev) saat testing.

🎯 Tujuan Belajar

Welcome to the Finale! Setelah melalui seluruh fase pembuatan arsitektur di tahap sebelumnya, mari kita bedah *What exactly happened* (Apa yang sesungguhnya terjadi). Pada materi ini kita akan:

  • Mengkaji ulang (*Review*) perbandingan drastis antara kode HTML statis (Tahap 1) dengan kode Blade + Livewire dinamis terbaru (Tahap 2-4).
  • Memahami kedahsyatan `wire:model` dalam menyambung input form dengan PHP secara tanpa kelim (*seamless*).
  • Mengganti metrik dummy di Dashboard menjadi nilai Query Eloquent asli.

🔍 1. Anatomi Perubahan: Form Payroll (Sebelum vs Sesudah)

Mari sandingkan perubahan rancangan UI Kalkulator Payroll yang awalnya "mati" tanpa nyawa di Tahap 1 melawan versi "hidup"-nya yang kita desain di Tahap 4.

❌ Tahap 1 (Statis - HTML Murni biasa) 
<!-- Dropdown Karyawan Mati (Hardcoded) -->
<select class="w-full rounded-md border-gray-300">
    <option disabled selected>-- Cari Karyawan --</option>
    <option>EMP-001 | Budi Santoso</option>
</select>

<!-- Input Nominal Mati -->
<input type="number" class="w-full rounded border-gray-300" placeholder="Rp 0">

<!-- Take Home Pay Hasil Angka Mati -->
<div class="text-3xl font-extrabold text-blue-600">
    Rp 0
</div>
✅ TAHAP 4 (Dinamis - Livewire Component Blade) 
<!-- Dropdown Dinamis mengambil referensi dari properti $employees dari DB -->
<select wire:model="employee_id" class="w-full rounded-md border-gray-300">
    <option value="">-- Cari Karyawan --</option>
    @foreach($employees as $emp)
        <option value="{{ $emp->id }}">{{ $emp->nik }} | {{ $emp->name }}</option>
    @endforeach
</select>

<!-- Input hidup diikat ke Backend, menembakkan Hook updated() saat diubah -->
<input wire:model.live="basic_salary" type="number" class="w-full rounded border-gray-300">

<!-- Take Home Pay tercetak otomatis merubah digit merespons state property PHP -->
<div class="text-3xl font-extrabold text-blue-600">
    Rp {{ number_format($net_salary, 0, ',', '.') }}
</div>

💡 Penjelasan: Apa fungsi `wire:model`?

Singkatnya: Jembatan Dua Arah (Two-Way Binding).

Ketika kamu memberikan atribut wire:model.live="basic_salary" pada input teks HTML, nilai yang diketik oleh pengguna akan langsung menyebrang jaringan internet menuju Variabel Public di PHP (public $basic_salary). Begitu juga sebaliknya, jika kamu memerintahkan PHP mengubah nilainya, angka di layar browser pengguna otomatis tergantikan tanpa kedipan layar (Page Reload).

📊 2. Finalisasi Metrik Halaman Dashboard

Pada file statis awal (static/dashboard.blade.php) kita mengisikan angka palsu. Sekarang kita ganti dengan data nyata dari database menggunakan Livewire component khusus Dashboard.

Langkah 1 — Buat Komponen Dashboard

Terminal 
php artisan make:livewire Dashboard

Langkah 2 — Isi Logika di app/Livewire/Dashboard.php

app/Livewire/Dashboard.php 
namespace App\Livewire;

use Livewire\Component;
use App\Models\Employee;
use App\Models\Payroll;

class Dashboard extends Component
{
    public function render()
    {
        // Gunakan kolom month_year (bukan created_at) agar filter periode konsisten
        $periodeBulanIni = \Carbon\Carbon::now()->locale('id')->isoFormat('MMMM YYYY');

        return view('livewire.dashboard', [
            'total_employee'      => Employee::count(),
            'total_payroll_month' => Payroll::where('month_year', $periodeBulanIni)->sum('net_salary'),
        ])->layout('layouts.app');
    }
}

Langkah 3 — Ubah tampilan di resources/views/livewire/dashboard.blade.php

resources/views/livewire/dashboard.blade.php — bagian card angka 
<!-- Angka palsu (dari mockup Tahap 1) diganti dinamis: -->
<p class="text-3xl font-bold text-gray-800">{{ $total_employee }} Orang</p>

<p class="text-3xl font-bold text-gray-800">Rp {{ number_format($total_payroll_month, 0, ',', '.') }}</p>

🔗 3. File routes/web.php Lengkap Project Payroll

Ini adalah isi routes/web.php final setelah semua komponen selesai dibuat (Tahap 2–4). Pastikan semua use statement ada di atas file:

routes/web.php — versi final 
use Illuminate\Support\Facades\Route;
use App\Livewire\Dashboard;
use App\Livewire\EmployeeManager;
use App\Livewire\PayrollCalculator;
use App\Livewire\PayrollHistory;
use App\Models\Payroll;
use Barryvdh\DomPDF\Facade\Pdf;

// Halaman utama → redirect ke dashboard jika sudah login
Route::get('/', function () {
    return redirect()->route('dashboard');
});

// Route auth bawaan Breeze (login, register, logout, dll)
require __DIR__ . '/auth.php';

// Route yang butuh login
Route::middleware('auth')->group(function () {

    // Dashboard — menggunakan Livewire component agar data real-time
    Route::get('/dashboard', Dashboard::class)->name('dashboard');

    // Karyawan
    Route::get('/karyawan', EmployeeManager::class)->name('employees.index');

    // Payroll
    Route::get('/payroll', PayrollCalculator::class)->name('payroll.calculator');
    Route::get('/riwayat-gaji', PayrollHistory::class)->name('payroll.history');

    // Cetak PDF slip gaji (Tahap 6)
    Route::get('/cetak-slip/{id}', function ($id) {
        $payroll = Payroll::with('employee')->findOrFail($id);
        $pdf = Pdf::loadView('pdf.slip-gaji', ['data' => $payroll]);
        return $pdf->stream('Slip_Gaji_' . $payroll->employee->nik . '.pdf');
    })->name('payroll.cetak');

});

🎉 Kesimpulan Tutorial Payroll

  • Kamu telah membangun aplikasi payroll nyata dengan TALL Stack (Tailwind, Alpine.js, Laravel 13, Livewire 3) — teknologi yang dipakai di industri.
  • Breeze mengurus seluruh auth (login, register, logout) tanpa menulis satu baris kode auth pun. Kamu fokus ke bisnis logic.
  • Satu file web.php dengan satu Route::middleware('auth')->group() cukup melindungi semua halaman sekaligus.
  • Ini baru permulaan — bisa dikembangkan lebih jauh: role/permission, export Excel, notifikasi email slip gaji, dll.
