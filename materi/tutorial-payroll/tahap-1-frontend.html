Tahap 1: Tampilan Frontend Aplikasi Payroll

🚀 Tutorial Proyek ⏱️ 45 menit 🎨 Praktik Desain UI
⚙️ Versi: 🔥 Laravel 13.x ⚡ Livewire 3.x 🐘 PHP 8.3+ 🎨 Tailwind CSS 3 Tahap ini fokus desain UI statis. Pastikan Tailwind CSS sudah ter-setup via Vite (default Laravel 11 + Breeze).

📌 Prasyarat

  • Konsep HTML Dasar & Tailwind CSS
  • Memahami letak file resources/views dalam folder project laravel.
  • Node.js sudah terinstall di komputer (cek: node -v). Tailwind akan dikompilasi di Tahap 2 — untuk tahap ini kita fokus dulu ke struktur Blade-nya.

🎯 Tujuan Belajar

Setelah menyelesaikan materi tahap 1 ini, kamu akan mampu:

  • Membangun *mockup* UI statis sebelum logika Backend ditulis (mencegah kebingungan saat menggabungkan logika)
  • Mengatur struktur halaman Dashboard yang rapi dan profesional menggunakan grid system bawaan Tailwind.
  • Membangun UI form kalkulasi dinamis untuk Master Karyawan dan Payroll tanpa menyentuh *database* terlebih dahulu.

🛠️ Persiapan: Buat Project Laravel Baru

Sebelum menulis satu baris tampilan pun, project Laravel-nya harus ada dulu. Buka terminal, arahkan ke folder web server kamu (contoh: C:\laragon\www untuk Laragon atau C:\xampp\htdocs untuk XAMPP), lalu jalankan perintah berikut:

Terminal — Buat Project Laravel Baru 
# Buat project Laravel baru bernama "payroll-app"
laravel new payroll-app

# Masuk ke folder project
cd payroll-app
📌

Belum punya installer Laravel?

Jalankan sekali saja: composer global require laravel/installer — setelah itu perintah laravel new bisa dipakai kapan saja.

Selanjutnya, konfigurasi koneksi database. Buka file .env di root folder project dan sesuaikan tiga baris ini:

.env 
DB_DATABASE=payroll_db
DB_USERNAME=root
DB_PASSWORD=
💡

Buat database-nya dulu!

Buka phpMyAdmin (via http://localhost/phpmyadmin), klik New, beri nama payroll_db, lalu klik Create. Tanpa ini, migration akan error.

Terakhir, jalankan server development Laravel agar kita bisa langsung melihat hasilnya di browser:

Terminal 
php artisan serve

Sekarang project sudah siap. Buka browser dan akses http://localhost:8000 — kamu akan melihat halaman welcome Laravel. Lanjut ke pembuatan tampilan!

🎨 Tahap 1: Tampilan Statis (Mockup Frontend)

Pada tahap awal ini, kita tidak akan menyentuh model database maupun Livewire controller terlebih dahulu. Tujuannya adalah untuk mendesain rupa antarmuka (*UI/UX*) sistem payroll kita.

Buat folder baru bernama static/ di dalam resources/views/ milik project laravel kamu, dan mari kita buat *mockup* setiap halamannya satu per satu.

Cara Mencoba Setup Secara Live:

Setelah membuat ke-tiga file Blade di atas, kamu dapat langsung menguji desainnya dengan mendaftarkan Route sementaranya agar bisa dibuka lewat URL.

Tambahkan kode berikut di bagian paling bawah pada file routes/web.php proyekmu:

routes/web.php 
// Mendaftarkan rute khusus untuk melihat Mockup HTML
Route::view('/statis-karyawan', 'static.employee');
Route::view('/statis-payroll', 'static.payroll');
Route::view('/statis-dashboard', 'static.dashboard');

Lalu, pastikan server sedang berjalan (php artisan serve) dan akses tautan ini di browsermu:

  • 👉 http://localhost:8000/statis-karyawan
  • 👉 http://localhost:8000/statis-payroll
  • 👉 http://localhost:8000/statis-dashboard

📋 Ringkasan Tahap 1

  • Pembuatan Wireframe & Desain Visual sangat krusial sebelum mengetik *backend code*. Ini mencegah perubahan bentuk tampilan ketika aplikasi sudah mulai memuat data besar.
  • Kami menggunakan @vite(['resources/css/app.css']) agar Tailwind yang dikompilasi secara responsif terhubung ke file view.
  • Class utilitas Tailwind sepert `grid-cols-2` sangat diandalkan untuk membuat panel sebelah-menyebelah yang lincah pada HP dan laptop (responsive).
