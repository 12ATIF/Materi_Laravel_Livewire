Tahap 4: Kalkulasi & Logika Payroll

🚀 Tutorial Proyek ⏱️ 50 menit 🧮 Praktik Lifecycle Hook
⚙️ Versi: 🔥 Laravel 13.x ⚡ Livewire 3.x 🐘 PHP 8.3+ Lifecycle hook di Livewire 3: mount(), updating(), updated(), hydrate(), dehydrate(). Method updatedNamaProperty() tetap berlaku untuk perubahan property tertentu.

📌 Prasyarat

  • Konsep Lifecycle Hooks Livewire (updated()).
  • Berhasil menyelesaikan Tahap 3 sehingga tabel Karyawan di *database* sudah terbentuk.

🎯 Tujuan Belajar

Materi tahap 4 ini adalah Core / Inti dari nama aplikasi kita. Kamu akan mampu:

  • Menggunakan Lifecycle Hook updated() untuk menciptakan kalkulator perhitungan otomatis layaknya Microsoft Excel (Gaji Pokok + Tunjangan - Potongan).
  • Membuat skema database untuk merekam riwayat penggajian karyawan per bulan.

📂 1. Tabel Database Penggajian (Payroll)

Kita memerlukan satu tabel lagi untuk merekam struk gaji yang nantinya akan dibayarkan setiap bulan kepada karyawan.

Terminal 
# Membuat Model Payroll beserta migrasinya
php artisan make:model Payroll -m

Buka file *_create_payrolls_table.php di folder `database/migrations` dan tambahkan struktur berikut:

*_create_payrolls_table.php 
public function up(): void
{
    Schema::create('payrolls', function (Blueprint $table) {
        $table->id();
        // Menyambungkan tabel ini dengan tabel employees (Foreign Key)
        $table->foreignId('employee_id')->constrained()->cascadeOnDelete();
        
        // Komponen angka - Decimal 15 digit dengan 2 angka di belakang koma untuk presisi Uang
        $table->decimal('basic_salary', 15, 2)->default(0);   // Gaji Pokok
        $table->decimal('allowance', 15, 2)->default(0);      // Total Tunjangan
        $table->decimal('deduction', 15, 2)->default(0);      // Total Potongan
        $table->decimal('net_salary', 15, 2)->default(0);     // Gaji Bersih Terakhir (Take Home Pay)
        
        $table->string('month_year');                         // Contoh: "April 2026"
        $table->timestamps();
    });
}

Buka app/Models/Payroll.php dan pasang $fillable serta Relasi Balik ke Employee:

app/Models/Payroll.php 
class Payroll extends Model
{
    protected $fillable = ['employee_id', 'basic_salary', 'allowance', 'deduction', 'net_salary', 'month_year'];

    // Setiap slip gaji ini adalah kepunyaan SATU karyawan.
    public function employee()
    {
        return $this->belongsTo(Employee::class);
    }
}

Jalankan php artisan migrate untuk memproses perintah ini ke tabel MySQL Anda.

🧮 2. Sihir Kalkulator Pada Livewire Component

Di Tahap 1 kita sudah mendesain static/payroll.blade.php di mana terdapat form Gaji Pokok, Tunjangan, dan Potongan. Tugas kita sekarang adalah membuat otak di baliknya sehingga Gaji Bersih otomatis terhitung saat angkanya diketik.

Terminal 
# Membuat komponen transaksi Gaji
php artisan make:livewire PayrollCalculator

Penggunaan Hook updated() untuk Agregasi:

Buka app/Livewire/PayrollCalculator.php:

app/Livewire/PayrollCalculator.php 
namespace App\Livewire;

use Livewire\Component;
use Illuminate\Validation\Rule;
use App\Models\Employee;
use App\Models\Payroll;

class PayrollCalculator extends Component
{
    // Input
    public ?int    $employee_id  = null;
    public int     $basic_salary = 0;
    public int     $allowance    = 0;
    public int     $deduction    = 0;
    public string  $month_year = '';  // user pilih sendiri: "April 2026"

    // Output (dihitung otomatis)
    public int $net_salary = 0;

    public function mount()
    {
        // Default bulan/tahun ke bulan ini (format Indonesia)
        $this->month_year = \Carbon\Carbon::now()->locale('id')->isoFormat('MMMM YYYY');
    }

    // Lifecycle Hook: hitung ulang THP setiap kali input angka berubah
    public function updated($field)
    {
        if (in_array($field, ['basic_salary', 'allowance', 'deduction'])) {
            // max(0, ...) mencegah THP negatif jika potongan melebihi pendapatan
            $this->net_salary = max(0, ($this->basic_salary + $this->allowance) - $this->deduction);
        }
    }

    public function savePayroll()
    {
        $this->validate([
            'employee_id'  => [
                'required',
                'exists:employees,id',
                // Cegah slip gaji duplikat: satu karyawan max 1 slip per periode
                Rule::unique('payrolls', 'employee_id')->where('month_year', $this->month_year),
            ],
            'basic_salary' => 'required|integer|min:1',
            'month_year'   => 'required|string',
        ], [
            'employee_id.required' => 'Silakan pilih karyawan terlebih dahulu.',
            'employee_id.exists'   => 'Karyawan tidak ditemukan di database.',
            'employee_id.unique'   => 'Karyawan ini sudah memiliki slip gaji untuk periode ' . $this->month_year . '.',
            'basic_salary.required' => 'Gaji pokok wajib diisi.',
            'basic_salary.min'     => 'Gaji pokok harus lebih dari 0.',
            'month_year.required'  => 'Periode gaji wajib diisi.',
        ]);

        Payroll::create([
            'employee_id'  => $this->employee_id,
            'basic_salary' => $this->basic_salary,
            'allowance'    => $this->allowance,
            'deduction'    => $this->deduction,
            'net_salary'   => $this->net_salary,
            'month_year'   => $this->month_year,
        ]);

        session()->flash('success', 'Slip gaji berhasil diterbitkan untuk ' . $this->month_year . '!');
        $this->reset(['employee_id', 'basic_salary', 'allowance', 'deduction', 'net_salary']);
        // Reset periode ke bulan sekarang — jangan panggil mount() secara manual!
        $this->month_year = \Carbon\Carbon::now()->locale('id')->isoFormat('MMMM YYYY');
    }

    public function render()
    {
        return view('livewire.payroll-calculator', [
            'employees' => Employee::orderBy('name', 'asc')->get(),
        ])->layout('layouts.app');
    }
}
🛡️

Kenapa ada Rule::unique di validasi employee_id?

Kita ingin mencegah satu karyawan memiliki dua slip gaji untuk periode yang sama. Dengan Rule::unique('payrolls', 'employee_id')->where('month_year', $this->month_year), Laravel akan cek: "Apakah kombinasi karyawan ini + periode ini sudah ada di database?" — jika sudah ada, form akan ditolak dengan pesan error sebelum data disimpan ganda.

📚 Kenapa updated() Sangat Krusial?

Pada PHP murni atau Controller Laravel biasa, untuk menghitung nilai otomatis saat user mengetik, kamu harus menulis JavaScript (event listener onkeyup). Di Livewire, cukup fungsi updated() di PHP — logika bisnis tetap di backend, tidak bisa diotak-atik via Inspect Element browser.

🎨 Blade View: payroll-calculator.blade.php

Sekarang buat tampilan form-nya. Ini adalah versi "hidup" dari mockup Tahap 1 — hubungkan setiap input ke property PHP lewat wire:model.live agar kalkulator langsung berreaksi saat angka diketik.

    <!-- Flash sukses -->

            <div class="mb-4">

            </div>

            <div class="mb-4">
                <label class="block text-sm font-medium text-gray-700">Tunjangan</label>
                <input type="number" wire:model.live="allowance" min="0" class="mt-1 block w-full rounded border-gray-300 shadow-sm" placeholder="0">
            </div>

            <div class="mb-6">
                <label class="block text-sm font-medium text-gray-700">Potongan (BPJS, Pajak, dll)</label>
                <input type="number" wire:model.live="deduction" min="0" class="mt-1 block w-full rounded border-gray-300 shadow-sm" placeholder="0">

                    <p class="text-xs text-gray-400">Gaji Pokok + Tunjangan - Potongan</p>

                <div class="text-3xl font-extrabold text-blue-600">

</div>
✨

Perhatikan wire:loading di Tombol Simpan

wire:loading.attr="disabled" otomatis menonaktifkan tombol saat request sedang diproses — mencegah user klik dua kali dan membuat slip gaji duplikat. wire:loading / wire:loading.remove mengganti teks tombol antara "Simpan" dan "Menyimpan...". Ini UX profesional dengan nol baris JavaScript!

🔗 Daftarkan Route Payroll Calculator

routes/web.php — tambahkan di dalam group auth 
use App\Livewire\PayrollCalculator;

Route::middleware('auth')->group(function () {
    // ... route karyawan dari Tahap 3 ...
    Route::get('/payroll', PayrollCalculator::class)->name('payroll.calculator');
});

Akses di browser: http://localhost:8000/payroll

📋 Ringkasan Tahap 4

  • Tabel payrolls punya foreign key ke employees dan kolom month_year yang bisa diisi manual — tidak lagi hardcoded English.
  • updated() hanya menghitung ulang saat field angka berubah (bukan semua field) — lebih efisien.
  • mount() dipakai untuk nilai default month_year dalam bahasa Indonesia via Carbon locale('id').
  • wire:loading di tombol simpan mencegah double-submit dan memberi feedback visual ke user.
