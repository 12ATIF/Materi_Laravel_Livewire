1.3 Instalasi Laravel & Livewire

📗 Minggu 1 ⏱️ 30 menit 🔧 Praktik

📌 Prasyarat

  • Sudah membaca 1.1 Pengenalan Laravel dan 1.2 MVC Sederhana
  • Sudah paham: Laravel = framework MVC, Composer = package manager
  • Komputer dengan Windows 10/11 (minimal RAM 4GB)

🎯 Tujuan Belajar

Setelah menyelesaikan materi ini, kamu akan mampu:

  • Menginstall Laragon sebagai environment development lokal
  • Membuat project Laravel baru via terminal
  • Menjalankan development server dan melihat hasilnya di browser
  • Menginstall Livewire ke dalam project Laravel
  • Mengatasi error umum saat instalasi

🛠️ Apa Saja yang Kita Butuhkan?

Sebelum mulai coding, kita perlu menyiapkan "bengkel" dulu. Berikut tool yang dibutuhkan:

🔧 Laragon
Web server lokal
+
📦 Composer
Package manager
+
🚀 Laravel
Framework PHP
+
⚡ Livewire
Extension interaktif
🏗️

Analogi: Bengkel Tukang

Laragon = meja kerja + alat-alat dasar (palu, gergaji, dll). Composer = toko material yang mengantarkan bahan ke bengkelmu. Laravel = blueprint rumah. Livewire = alat listrik canggih untuk fitur spesial.

1️⃣ Install Laragon

1

Download Laragon

Buka browser, pergi ke laragon.org/download. Pilih versi "Full" (sudah termasuk PHP 8, MySQL, Apache, Composer, Git).

2

Install Laragon

Jalankan file installer yang sudah didownload. Ikuti langkah default-nya:

  • Pilih lokasi install: C:\laragon (biarkan default)
  • Centang semua opsi yang ditawarkan
  • Klik Install dan tunggu sampai selesai
3

Jalankan Laragon

Buka Laragon, klik tombol "Start All". Pastikan Apache dan MySQL menyala (indikator hijau).

📌

Kenapa Laragon dan Bukan XAMPP?

Fitur Laragon XAMPP
Composer bawaan ✅ Sudah termasuk ❌ Harus install sendiri
Git bawaan ✅ Sudah termasuk ❌ Harus install sendiri
Auto Virtual Host ✅ Otomatis ❌ Harus setting manual
Terminal bawaan ✅ Cmder terintegrasi ⚠️ Harus pakai CMD/PowerShell
Quick Create Laravel ✅ Klik kanan → Create Laravel ❌ Harus manual via terminal

2️⃣ Buat Project Laravel Baru

Ada 2 cara untuk membuat project Laravel baru. Pilih yang paling nyaman:

Cara 1: Via Terminal (Direkomendasikan)

1

Buka Terminal Laragon

Klik kanan di Laragon → pilih "Terminal". Akan terbuka Cmder (terminal bawaan Laragon).

2

Masuk ke Folder www

Ketik perintah berikut untuk masuk ke folder project:

Terminal Laragon 
# Masuk ke folder www (tempat semua project)
cd C:\laragon\www
3

Buat Project Laravel

Jalankan perintah Composer untuk membuat project baru. Proses ini akan download semua file Laravel dari internet (±5 menit tergantung koneksi).

Terminal Laragon 
# Buat project Laravel dengan nama "perpustakaan"
composer create-project laravel/laravel perpustakaan

# Tunggu... ini akan download sekitar 80MB file
# Kalau sukses, akan muncul "Application key set successfully."
🔍

Penjelasan Perintah

  • composer — Panggil package manager Composer
  • create-project — Perintah untuk membuat project baru
  • laravel/laravel — Nama package Laravel dari Packagist (toko online package PHP)
  • perpustakaan — Nama folder project-mu. Bisa diganti sesuai keinginan!

Cara 2: Via Laragon GUI (Cepat & Mudah)

1

Klik Kanan di Laragon

Di window Laragon, klik kanan → Quick app → Laravel

2

Masukkan Nama Project

Ketik nama project: perpustakaan. Klik OK dan tunggu sampai selesai.

💡

Mana yang Lebih Baik?

Kedua cara hasilnya sama. Tapi via terminal lebih direkomendasikan karena kamu akan sering pakai terminal untuk menjalankan Artisan commands di materi-materi selanjutnya. Lebih baik terbiasa dari sekarang!

3️⃣ Jalankan Development Server

1

Masuk ke Folder Project

Di terminal, masuk ke folder project yang baru dibuat:

Terminal 
# Masuk ke folder project
cd perpustakaan

# Jalankan server development Laravel
php artisan serve
🔍

Apa yang Terjadi?

  • php artisan serve — Menjalankan web server bawaan Laravel di localhost:8000
  • Terminal akan menampilkan: Server running on [http://127.0.0.1:8000]
  • Jangan tutup terminal ini! Server berjalan selama terminal terbuka
2

Buka di Browser

Buka browser (Chrome/Firefox), ketik: http://localhost:8000

Kalau muncul halaman Welcome Laravel — 🎉 SELAMAT! Instalasi berhasil!

🌐

Alternatif: Pakai Pretty URL Laragon

Kalau Apache di Laragon sudah running, kamu juga bisa akses project tanpa php artisan serve. Cukup buka: http://perpustakaan.test — Laragon otomatis membuat virtual host!

4️⃣ Install Livewire

Sekarang kita install Livewire — package yang membuat Laravel bisa interaktif tanpa JavaScript!

1

Buka Terminal Baru

Buka tab terminal baru (jangan tutup yang lama!). Pastikan masih di folder perpustakaan.

Terminal (tab baru) 
# Pastikan masih di folder project
cd C:\laragon\www\perpustakaan

# Install Livewire via Composer
composer require livewire/livewire

# Cek apakah Livewire berhasil terinstall
php artisan livewire:version
🔍

Penjelasan Perintah

  • composer require — Perintah untuk menambahkan package baru ke project
  • livewire/livewire — Nama package Livewire dari Packagist
  • php artisan livewire:version — Cek versi Livewire yang terinstall

Kalau berhasil, terminal akan menampilkan versi seperti: Livewire: v3.x.x

2

Tambahkan Livewire ke Layout Blade

Supaya Livewire bisa bekerja, kita perlu menambahkan script-nya ke file layout. Buka file resources/views/welcome.blade.php dan pastikan ada tag berikut:

resources/views/welcome.blade.php 
<html>
<head>
    <!-- Tambahkan ini di dalam <head> -->
    @livewireStyles
</head>
<body>

    <!-- Konten halaman di sini -->

    <!-- Tambahkan ini sebelum </body> -->
    @livewireScripts
</body>
</html>
💡

Ingat Aturannya!

@livewireStyles selalu di <head> (untuk CSS Livewire)
@livewireScripts selalu sebelum </body> (untuk JavaScript Livewire)
Tanpa kedua tag ini, komponen Livewire tidak akan bekerja!

🔧 Troubleshooting — Kalau Ada Error

Tenang, error saat instalasi itu normal. Berikut solusi untuk masalah yang sering muncul:

⚠️ Error yang Sering Terjadi

❌ "composer is not recognized as an internal command"

Artinya: Composer belum terinstall atau belum dikenali oleh sistem.

✅ Solusi: Pastikan Laragon sudah dijalankan (klik Start All). Buka terminal via Laragon, JANGAN via CMD Windows langsung.

❌ "Your requirements could not be resolved..."

Artinya: Versi PHP-mu mungkin terlalu lama untuk Laravel terbaru.

✅ Solusi: Update PHP di Laragon. Buka Menu → Tools → Change PHP → pilih versi 8.1 atau lebih baru.

❌ "Failed to download laravel/laravel" atau koneksi timeout

Artinya: Koneksi internet bermasalah atau toko Packagist sedang lambat.

✅ Solusi: Cek koneksimu. Kalau wifi sekolah lambat, coba pakai hotspot HP. Atau tambahkan --prefer-dist di perintah.

❌ "Port 8000 already in use"

Artinya: Sudah ada program lain yang memakai port 8000.

✅ Solusi: Ganti port dengan: php artisan serve --port=8080 lalu buka localhost:8080.

📋 Ringkasan Materi

  • Laragon = environment development all-in-one untuk Windows
  • Buat project: composer create-project laravel/laravel nama-project
  • Jalankan server: php artisan serve → buka localhost:8000
  • Install Livewire: composer require livewire/livewire
  • Jangan lupa tambahkan @livewireStyles dan @livewireScripts ke layout

✅ Checklist Selesai Instalasi

Pastikan semua item berikut sudah tercyeklist sebelum lanjut ke materi berikutnya:

  • ✓ Laragon terinstall dan bisa Start All tanpa error
  • ✓ Bisa buka terminal dari Laragon
  • ✓ Project Laravel berhasil dibuat (ada folder perpustakaan di C:\laragon\www\)
  • ✓ php artisan serve berjalan dan bisa diakses di browser
  • ✓ Livewire terinstall (perintah php artisan livewire:version menampilkan versi)

📝 Latihan

🟢 Level 1 — Praktik Dasar

  1. Install Laragon di komputermu dan screenshot tampilan ketika sudah Start All.
  2. Buat project Laravel baru dengan nama "belajar-livewire". Screenshot terminal ketika sukses.
  3. Jalankan php artisan serve lalu screenshot halaman Welcome Laravel di browser.

🔵 Level 2 — Eksplorasi

  1. Install Livewire ke project-mu. Jalankan php artisan livewire:version dan catat versinya.
  2. Coba jalankan php artisan list — ini akan menampilkan semua perintah Artisan yang tersedia. Berapa total perintah yang ada?
  3. Buka folder project di File Explorer (C:\laragon\www\perpustakaan). Bandingkan struktur folder dengan yang kita pelajari di materi MVC. Apakah folder-folder Model, View, Controller ada di posisi yang benar?

🟣 Level 3 — Tantangan

  1. Buka file .env di root project. File ini berisi konfigurasi project. Cari baris yang mengatur nama aplikasi (APP_NAME) dan ganti menjadi "Perpustakaan SMK". Refresh browser — apa yang berubah?
  2. Coba buat project kedua dengan nama berbeda. Apakah keduanya bisa jalan bersamaan? Bagaimana caranya? (Petunjuk: gunakan port berbeda)
