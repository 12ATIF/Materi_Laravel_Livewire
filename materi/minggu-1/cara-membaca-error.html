1.7 🐛 Cara Membaca Error (Skill Wajib Programmer)

📗 Minggu 1 ⏱️ 35 menit 🛠️ Skill Praktis
⚙️ Versi: 🔥 Laravel 11.x ⚡ Livewire 3.x 🐘 PHP 8.2+ Format halaman error berbeda antar versi Laravel, tapi cara membacanya sama — fokus di pesan utama, file, dan baris error.
🚨

Cek Ini Dulu Sebelum Mulai: APP_DEBUG=true

Laravel punya dua mode tampilan error. Kalau kamu tidak bisa melihat detail error (cuma layar putih atau tulisan "Server Error"), kemungkinan besar setting ini yang salah.

Buka file .env di root project, pastikan:

APP_DEBUG=true   # ← harus true saat development/belajar
APP_ENV=local    # ← bukan "production"

Setelah ubah .env, jalankan php artisan config:clear di terminal agar perubahan terbaca. Di production (server online), wajib ganti ke APP_DEBUG=false agar kode internal tidak bocor ke publik.

👨‍🏫

Pesan dari Guru

Pemula sering panik saat lihat layar merah penuh tulisan aneh. Itu wajar. Tapi rahasianya: error itu teman, bukan musuh. Error memberitahumu persis di mana masalahnya. Programmer profesional tidak menghindari error — mereka membacanya. Setelah materi ini, kamu akan punya skill yang membuat 80% bug bisa kamu pecahkan sendiri tanpa nanya ke siapa pun.

🎯 Tujuan Belajar

Setelah menyelesaikan materi ini, kamu akan mampu:

  • Membaca anatomi halaman error Laravel (Ignition / Whoops)
  • Memahami stack trace dan menemukan baris kode yang bermasalah
  • Mengenali 6 jenis error paling umum dan cara mengatasinya
  • Menggunakan tools debug: dd(), dump(), dan Log
  • Mencari solusi error dengan Google secara efektif
  • Membaca error khusus Livewire di console browser

🤔 Kenapa Programmer Pemula Takut Error?

Saat baru belajar, layar merah penuh tulisan teknis bisa terlihat menakutkan. Tapi sebenarnya:

💡

Analogi: Dokter & Pasien

Bayangkan kamu ke dokter karena sakit. Dokter tidak akan marah kalau kamu sakit — itu tugasnya. Dokter akan baca gejala, lalu kasih obat. Begitu juga dengan error: itu "gejala" kode kamu. Tugasmu adalah baca gejalanya, baru kasih "obat" (perbaiki kode).

Mindset yang Harus Dimiliki

❌ Mindset Salah

  • "Aduh error lagi, gue payah!"
  • Langsung tutup browser, panik
  • Asal copy-paste kode dari Stack Overflow
  • Tanya ke teman tanpa baca pesan error

✅ Mindset Benar

  • "Oke, ada gejala. Apa yang error katakan?"
  • Baca pelan-pelan dari atas
  • Catat: pesan utama, file, baris
  • Coba pahami dulu, baru cari solusi

4 Bagian Penting yang HARUS Dibaca

No Bagian Apa yang Dicari?

⚠️

Tips Pertama yang WAJIB

Baca pesan utamanya dulu! Banyak pemula langsung scroll ke stack trace, padahal 90% jawabannya ada di kalimat pertama. Latihlah dirimu untuk baca dari atas, pelan-pelan.

🔍 Membaca Stack Trace

Stack trace = riwayat fungsi yang saling memanggil sampai akhirnya kena error. Bacanya dari atas ke bawah, tapi yang paling penting adalah baris pertama yang menunjuk ke kode-mu (bukan kode framework Laravel/Livewire).

🏛️

Analogi: Petunjuk Polisi

Stack trace itu seperti polisi yang merunut kasus: "Sebelum kejadian X, ada apa? Sebelum itu lagi, ada apa?". Tugasmu adalah cari titik pertama kali kode kamu muncul di rangkaian itu. Itulah TKP yang sebenarnya.

Contoh Membaca Stack Trace

stack-trace-example.txt 
vendor/laravel/framework/src/Illuminate/Database/Connection.php:712  ← framework
vendor/laravel/framework/src/Illuminate/Database/Eloquent/Model.php:1234 ← framework
vendor/laravel/framework/src/Illuminate/Database/Eloquent/Builder.php:567 ← framework
app/Livewire/BookList.php:33  ← INILAH KODEMU! Mulai cari di sini.
vendor/livewire/livewire/src/...                  ← framework

Aturan emas: Abaikan baris yang berisi vendor/. Cari baris pertama yang berisi app/ — itulah kode kamu yang harus diperbaiki.

🚨 5 Error Paling Umum & Cara Bacanya

1. Class Not Found

Error: Class "App\Models\Buku" not found

Artinya: Laravel tidak bisa menemukan class yang kamu panggil.

Penyebab umum:

  • Salah ketik nama class (misal: Buku vs buku)
  • Lupa use App\Models\Buku; di atas file
  • File class belum dibuat / typo nama file
  • Setelah pindah file, lupa jalankan composer dump-autoload

Solusi cepat:

terminal 
composer dump-autoload

2. Undefined Variable

ErrorException: Undefined variable $books

Artinya: Kamu pakai variabel $books di view, tapi tidak pernah dikirim dari controller/component.

Solusi: Cek apakah variabel sudah di-passing:

app/Livewire/BookList.php 
public function render()
{
    return view('livewire.book-list', [
        'books' => Book::all(),  // ← pastikan key 'books' ada di sini
    ]);
}

3. Method Not Found / BadMethodCallException

BadMethodCallException: Call to undefined method
App\Livewire\BookList::saveBuku()

Artinya: Di Blade kamu pakai wire:click="saveBuku", tapi method saveBuku() tidak ada di class component.

Cek:

  • Apakah method-nya public function? (private/protected tidak bisa dipanggil dari Blade)
  • Apakah ejaan-nya sama persis? saveBuku ≠ savebuku ≠ save_buku
  • Apakah method ada di file yang benar?

4. Database Error (Table Not Found)

SQLSTATE[42S02]: Base table or view not found:
1146 Table 'db_perpus.bukus' doesn't exist

Artinya: Tabel bukus belum dibuat di database.

Solusi:

terminal 
# Pastikan migration sudah dijalankan
php artisan migrate

# Kalau sudah pernah migrate tapi mau ulang dari nol:
php artisan migrate:fresh
⚠️

Hati-hati migrate:fresh

Perintah ini menghapus semua data dan membuat ulang tabel. Jangan dipakai di database production!

5. Syntax Error / ParseError

ParseError: syntax error, unexpected token "}"
in app/Livewire/BookList.php on line 42

Artinya: Ada salah ketik di kode PHP — biasanya kurang atau kelebihan {, }, ;, atau ".

Tips menemukannya:

  • Buka file yang disebut, pergi ke baris yang disebut dan beberapa baris sebelumnya
  • Pakai editor (VS Code) — biasanya kasih warning warna sebelum error
  • Cek kurung kurawal apakah seimbang (jumlah { = jumlah })
  • Pastikan setiap statement diakhiri ;

6. Error 419 — Page Expired

419 | PAGE EXPIRED

Artinya: Session/CSRF token sudah kadaluwarsa. Ini sangat sering muncul di form Livewire setelah tab browser dibiarkan idle beberapa menit.

Bukan berarti kodenya salah! Ini fitur keamanan Laravel (CSRF protection).

Solusi:

  • Refresh halaman (F5), lalu isi ulang form dan submit
  • Kalau sering terjadi: cek SESSION_LIFETIME di .env (defaultnya 120 menit)
  • Pastikan layout Blade kamu punya @livewireScripts — Livewire sudah handle CSRF otomatis asal tag ini ada

🔄 Solusi "Misterius": Clear Cache

Kadang kamu sudah betulkan kode, tapi error masih muncul. Atau sebaliknya — kode salah tapi tidak ada error. Itu biasanya karena Laravel menyimpan cache dari konfigurasi/view/route lama.

💊

"Obat Mujarab" Laravel

Sebelum panik, coba jalankan perintah ini dulu di terminal:

terminal (di folder project) 
# Hapus semua cache sekaligus (paling praktis)
php artisan optimize:clear

# Atau satu per satu kalau mau spesifik:
php artisan config:clear   # cache konfigurasi .env
php artisan view:clear     # cache tampilan Blade
php artisan route:clear    # cache routing
php artisan cache:clear    # cache aplikasi
Situasi Cache yang Diclear
Ubah .env tapi tidak terbaca config:clear
Ubah Blade tapi tampilan tidak berubah view:clear
Tambah route baru tapi 404 route:clear
Tidak tahu masalahnya apa optimize:clear (hapus semua)

🛠️ Tools Debug Wajib: dd, dump, Log

Kadang error tidak langsung muncul, tapi data yang dihasilkan kode salah. Saat itulah kamu butuh "melihat" isi variabel saat program berjalan.

1. dd() — Dump and Die

Fungsi paling sering dipakai. Kepanjangannya: "Dump and Die" — tampilkan isi variabel, lalu hentikan program.

app/Livewire/BookList.php 
public function simpan()
{
    $books = Book::all();

    dd($books);  // ← lihat isinya, lalu STOP

    // kode di bawah ini TIDAK akan jalan
    return redirect()->route('home');
}

2. dump() — Dump Tanpa Berhenti

Sama seperti dd(), tapi program tetap lanjut. Berguna kalau kamu mau lihat beberapa variabel dalam alur yang sama.

dump($user);    // tampilkan, lanjut
dump($buku);    // tampilkan, lanjut
dump($total);   // tampilkan, lanjut

3. Log::info() — Tulis ke File Log

Untuk debug yang lebih serius (terutama di Livewire yang request-nya bolak-balik), pakai Log. Pesan akan ditulis ke file storage/logs/laravel.log.

use Illuminate\Support\Facades\Log;

Log::info('User klik tombol simpan', [
    'user_id' => auth()->id(),
    'data' => $this->form,
]);

Lalu di terminal:

terminal (di folder project) 
php artisan pail
# atau, cara lama:
tail -f storage/logs/laravel.log
Tool Kapan Pakai? Output di mana?
dd() Cek nilai sekali, hentikan eksekusi Browser
dump() Cek beberapa nilai, lanjutkan eksekusi Browser
Log::info() Debug Livewire / proses background File storage/logs/laravel.log

⚡ Membaca Error Khusus Livewire

Livewire punya tempat error yang berbeda dari Laravel biasa, karena banyak hal terjadi via JavaScript (di browser). Kamu harus cek 2 tempat:

🌐 Browser DevTools (F12)

  • Tab Console — error JavaScript & Livewire
  • Tab Network — lihat request livewire/update

🖥️ Layar Browser

  • Halaman error Laravel (Ignition)
  • Untuk error PHP di sisi server

Cara Buka Console Browser

  1. Tekan F12 di Chrome / Firefox / Edge
  2. Pilih tab Console
  3. Lihat tulisan merah di sana saat klik tombol Livewire

Error Livewire yang Sering Muncul

Console Browser 
[Livewire] Component not found: BookList
→ Lupa @livewireScripts di layout

[Livewire] Property [$bukuTitle] not found on component
→ Property belum dideklarasi di class component

[Livewire] Method [hapus] not found on component
→ Method belum ada / typo / bukan public

Menampilkan Error Validasi di Blade: @error

Ini bukan error yang bikin layar merah, tapi pesan error validasi yang ditampilkan ke user. Pemula sering bingung kenapa validasi sudah jalan tapi pesan tidak muncul di form.

resources/views/livewire/form-buku.blade.php 
<input type="text" wire:model="judul">

@error('judul')
    <p class="text-red-500 text-sm">{{ $message }}</p>
@enderror
📌

@error('nama_field') akan menampilkan $message berisi pesan validasi untuk field tersebut. Kalau tidak ada error untuk field itu, blok ini otomatis disembunyikan. Detail cara pakai validasi dibahas di materi Validasi →

🔎 Cara Google Error dengan Efektif

80% error yang kamu hadapi sudah pernah dialami orang lain. Skill mencari di Google adalah skill yang membedakan programmer pemula dengan menengah.

Rumus Search yang Benar

🎯

Format: [framework] [versi] [pesan error inti]

Contoh:

  • ✅ laravel 11 sqlstate 42s02 table not found
  • ✅ livewire 3 component not found
  • ✅ laravel class not found after rename
  • ❌ laravel error gimana cara fix (terlalu umum)
  • ❌ kenapa kode saya tidak jalan (mustahil dijawab)

Tips Tambahan Saat Search

  • Hapus bagian unik dari error sebelum di-search. Misal nama file/path spesifik project — itu cuma muncul di komputermu, hasil search akan kosong.
  • Tambahkan site:stackoverflow.com untuk cari hanya di Stack Overflow.
  • Cari di GitHub Issues repo Laravel/Livewire — banyak bug-versi yang dibahas maintainer langsung.
  • Cek tanggal jawaban! Jawaban tahun 2018 mungkin tidak relevan untuk Laravel 11.

🎯 Latihan: Baca Error Berikut

Coba kerjakan latihan ini sebelum lihat jawaban — ini melatih otakmu mengenali pola error.

🟢 Latihan 1 — Identifikasi

Error berikut bilang apa? Apa kira-kira penyebabnya?

Class "App\Livewire\BookListed" not found
at app/Providers/AppServiceProvider.php:18

Pertanyaan:

  1. Class apa yang tidak ditemukan?
  2. Di file mana error terjadi?
  3. Apa kemungkinan penyebabnya?
💡 Lihat Kunci Jawaban
  1. Class App\Livewire\BookListed tidak ditemukan.
  2. Error terjadi di app/Providers/AppServiceProvider.php baris 18 (tempat class itu didaftarkan).
  3. Kemungkinan penyebab: (a) nama file component adalah BookList.php tapi dipanggil BookListed — typo, (b) file component belum dibuat sama sekali, (c) file sudah ada tapi di folder yang salah. Solusi: cek isi AppServiceProvider.php baris 18, pastikan nama class dan nama file sama persis (case-sensitive).

🟡 Latihan 2 — Diagnosa

Berikut error di console browser saat klik tombol di Livewire:

Livewire: Method [simpanBuku] not found on
component: [book-form]

Tugas: Sebutkan 3 kemungkinan penyebab dan cara mengeceknya.

💡 Lihat Kunci Jawaban
  1. Method belum ada — Buka app/Livewire/BookForm.php, cari apakah ada public function simpanBuku().
  2. Typo nama method — Mungkin di Blade ditulis wire:click="simpanBuku" tapi di PHP ditulis simpanBuку (huruf besar/kecil berbeda) atau saveBuku. PHP case-sensitive untuk nama method.
  3. Method tidak public — Livewire hanya bisa memanggil method yang berstatus public. Kalau private function simpanBuku() atau protected, tidak bisa dipanggil dari Blade.

🟣 Latihan 3 — Skenario Nyata

Kamu sedang membuat form Tambah Buku. Setelah klik Simpan:

  • Tidak ada pesan error di layar
  • Tapi data tidak masuk ke database
  • Halaman cuma refresh, kembali ke form kosong

Tugas:

  1. Tools debug apa yang akan kamu pakai? (dd, dump, atau Log?)
  2. Di tahap mana saja kamu akan pasang debug? (sebelum simpan? sesudah simpan?)
  3. Apa yang akan kamu lihat untuk memastikan data benar-benar masuk?
💡 Lihat Kunci Jawaban
  1. Tools: Pakai dd() untuk menghentikan eksekusi dan melihat nilai variabel di titik tertentu. Karena tidak ada error, kita perlu "mengintip" data saat program berjalan.
  2. Titik debug:
    • Di awal method simpan(): dd($this->judul, $this->pengarang) — apakah data dari form sudah terbaca?
    • Setelah Book::create([...]): dd('berhasil disimpan') — apakah baris save dieksekusi?
  3. Verifikasi: Cek langsung di database via phpMyAdmin / Laragon DB, apakah baris baru muncul. Atau pakai dd(Book::latest()->first()) setelah create untuk lihat data terakhir yang tersimpan.

Penyebab paling umum di skenario ini: property wire:model tidak cocok dengan nama property di class, atau ada validasi yang gagal tanpa menampilkan pesan error.

📝 Rangkuman

  • Error itu teman — mereka memberitahu di mana masalahnya, bukan menghakimi.
  • 4 bagian halaman error: pesan, file & baris, kode highlight, stack trace.
  • Di stack trace, abaikan vendor/ — fokus ke app/.
  • Pastikan APP_DEBUG=true di .env agar detail error terlihat saat belajar.
  • 6 error umum: Class not found, Undefined variable, Method not found, Table not found, Syntax error, 419 Page Expired.
  • Error tidak muncul walaupun kode sudah benar? Jalankan php artisan optimize:clear.
  • Tools debug: dd() (lihat & stop), dump() (lihat & lanjut), Log::info() (catat ke file).
  • Livewire = cek 2 tempat: halaman browser & Console DevTools (F12).
  • Google dengan format: [framework] [versi] [pesan inti error].
