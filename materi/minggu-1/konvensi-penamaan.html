1.4 📏 Konvensi Penamaan Laravel

📗 Minggu 1 ⏱️ 30 menit 📖 Teori + Praktik
⚙️ Versi: 🔥 Laravel 11.x ⚡ Livewire 3.x 🐘 PHP 8.2+ Konvensi ini berlaku sama di semua versi Laravel modern. Mengikutinya adalah syarat agar fitur autoloading, route model binding, dan Eloquent bekerja otomatis.
👨‍🏫

Kenapa Materi Ini Penting Banget?

Dari pengalaman mengajar, 50% pertanyaan pemula bisa dijawab dengan satu kalimat: "Nama file/class/tabel kamu tidak sesuai konvensi Laravel." Laravel itu "magic" — banyak hal bekerja otomatis, tapi hanya kalau kamu mengikuti aturan penamaannya. Satu huruf besar yang salah bisa bikin fitur tidak jalan. Hafalkan tabel di halaman ini.

🎯 Tujuan Belajar

Setelah menyelesaikan materi ini, kamu akan mampu:

  • Memahami pola PascalCase, camelCase, snake_case, kebab-case dan kapan memakainya di Laravel
  • Menyebutkan konvensi nama untuk Model, Migration, Controller, View, Route
  • Memahami hubungan otomatis antara nama Model ↔ Tabel database
  • Menerapkan konvensi penamaan Livewire Component
  • Menghindari bug kelas-kelas "nama salah" yang sering menjebak pemula

📐 4 Format Penulisan Nama

Sebelum masuk ke aturan Laravel, kenali dulu 4 format penulisan yang sering dipakai:

Format Contoh Ciri Khas
PascalCase BookController, UserProfile Setiap kata diawali huruf BESAR, tanpa pemisah
camelCase bookTitle, getUserData Kata pertama huruf kecil, kata berikutnya huruf BESAR
snake_case book_title, created_at Semua huruf kecil, kata dipisah underscore _
kebab-case book-list, user-profile Semua huruf kecil, kata dipisah tanda minus -
💡

Cara Menghafal

  • PascalCase = untuk nama Class (PHP)
  • camelCase = untuk nama variable dan method di PHP
  • snake_case = untuk database (kolom, tabel), nama file migration
  • kebab-case = untuk URL dan nama Livewire di Blade

🏛️ Konvensi Lengkap Laravel

1. Model Eloquent

Yang Dibuat Format Contoh
Nama Class Model PascalCase, Singular Book, User, BookCategory
Nama File Model PascalCase + .php Book.php, BookCategory.php
Nama Tabel Database (otomatis) snake_case, Plural books, users, book_categories
🔁

Hubungan Otomatis: Model ↔ Tabel

Laravel secara otomatis mengubah nama Model ke nama tabel. Kamu tidak perlu menulis nama tabel di Model selama mengikuti konvensi.

Konversi otomatis Laravel 
Book         → tabel: books
User         → tabel: users
BookCategory → tabel: book_categories
StockItem   → tabel: stock_items

// Kalau tidak ikut konvensi, harus tulis manual di Model:
class Buku extends Model {
    protected $table = 'books'; // ← terpaksa tulis manual
}
⚠️

Hindari Nama Model Berbahasa Indonesia

Model bernama Buku akan mencari tabel bukus (bukan bukus yang valid). Model bernama Karyawan akan mencari karyawans. Laravel pluralize dalam bahasa Inggris. Rekomendasi: gunakan nama bahasa Inggris untuk Model (Book, Employee) agar aturan plural bekerja dengan benar.

2. Migration

Yang Dibuat Format Contoh
Nama File Migration timestamp_action_table.php (snake_case) 2024_01_01_000000_create_books_table.php
Nama Kolom Tabel snake_case book_title, published_at, author_id
Kolom Foreign Key nama_model_id (snake_case) user_id, category_id, book_id
terminal 
# Artisan akan beri nama file yang benar secara otomatis
php artisan make:migration create_books_table
php artisan make:migration add_cover_to_books_table

3. Controller

Yang Dibuat Format Contoh
Nama Class Controller PascalCase + Controller BookController, UserController
Nama File PascalCase + Controller.php BookController.php
Nama Method CRUD camelCase (ikut konvensi RESTful) index, create, store, show, edit, update, destroy

4. View (Blade)

Yang Dibuat Format Contoh
Nama File View kebab-case atau snake_case + .blade.php book-list.blade.php, create-book.blade.php
Folder View kebab-case atau snake_case, plural books/, user-profiles/
Memanggil view di PHP dot notation untuk folder view('books.book-list')
app/Http/Controllers/BookController.php 
// File ada di: resources/views/books/book-list.blade.php
return view('books.book-list'); // ← titik = pemisah folder

5. Route

Yang Dibuat Format Contoh
URL path kebab-case, plural untuk resource /books, /book-categories, /user-profiles
Nama route (->name()) dot notation + snake_case books.index, books.create, books.store
routes/web.php 
Route::get('/books', [BookController::class, 'index'])->name('books.index');
Route::get('/books/create', [BookController::class, 'create'])->name('books.create');

// Cara akses nama route di Blade:
<a href="{{ route('books.index') }}">Daftar Buku</a>

⚡ Konvensi Khusus Livewire

Livewire punya konvensi sendiri. Ini yang paling sering bikin bingung karena satu component punya 3 nama berbeda untuk 3 tujuan berbeda.

Konteks Format Contoh untuk component "BookList"
Nama Class PHP PascalCase BookList
Nama File PHP PascalCase + .php app/Livewire/BookList.php
Nama File Blade kebab-case + .blade.php resources/views/livewire/book-list.blade.php
Tag di Blade (dipanggil) <livewire:kebab-case /> <livewire:book-list />
terminal — buat Livewire component 
php artisan make:livewire BookList

Perintah di atas otomatis membuat 2 file:

app/Livewire/BookList.php              ← class PHP (PascalCase)
resources/views/livewire/book-list.blade.php  ← view Blade (kebab-case)
🚨

Jangan Buat File Manual!

Selalu gunakan php artisan make:livewire NamaComponent. Kalau buat file manual, sangat mudah salah nama. Artisan yang tahu persis format yang benar untuk kedua file sekaligus.

Konvensi Nama Livewire Untuk Kasus Nested

php artisan make:livewire Admin/BookManager
app/Livewire/Admin/BookManager.php
resources/views/livewire/admin/book-manager.blade.php

// Dipanggil di Blade dengan:
<livewire:admin.book-manager />   ← titik untuk subfolder

🔧 Konvensi Property & Method di PHP

Yang Dibuat Format Contoh
Nama property (variabel di class) camelCase $bookTitle, $isModalOpen, $selectedId
Nama method camelCase saveBook(), openModal(), deleteItem()
Nama konstanta SCREAMING_SNAKE_CASE MAX_UPLOAD_SIZE, DEFAULT_ROLE
app/Livewire/BookForm.php 
class BookForm extends Component   ← PascalCase (nama class)
{
    public $bookTitle = '';       ← camelCase (property)
    public $authorName = '';
    public $isModalOpen = false;

    public function saveBook()    ← camelCase (method)
    {
        Book::create([
            'book_title' => $this->bookTitle,  ← snake_case (kolom DB)
            'author_name' => $this->authorName,
        ]);
    }
}

📋 Tabel Ringkasan Lengkap

Cetak atau simpan tabel ini — ini adalah referensi cepat yang akan sering kamu lihat:

Komponen Laravel Format Nama Contoh
Model class PascalCase, Singular Book, UserProfile
Tabel database (otomatis) snake_case, Plural books, user_profiles
Kolom database snake_case book_title, created_at
Foreign key snake_case + _id user_id, category_id
Controller class PascalCase + Controller BookController
Livewire class PascalCase BookList, CreateBook
Livewire blade file kebab-case book-list.blade.php
Livewire tag di Blade <livewire:kebab-case /> <livewire:book-list />
View file kebab-case + .blade.php book-list.blade.php
URL path kebab-case, Plural /books, /book-categories
Nama route dot notation books.index, books.store
Property class (PHP) camelCase $bookTitle, $isOpen
Method class (PHP) camelCase saveBook(), deleteItem()
wire:model binding camelCase (cocok dengan property) wire:model="bookTitle"

🐛 Kesalahan Penamaan yang Sering Terjadi

❌ Salah Model: Buku → Laravel cari tabel "bukus"
✅ Benar Model: Book → Laravel cari tabel "books"
❌ Salah Livewire class: bookList → tidak valid (bukan PascalCase)
✅ Benar Livewire class: BookList → valid
❌ Salah <livewire:BookList /> → tidak bekerja
✅ Benar <livewire:book-list /> → bekerja
❌ Salah Kolom DB: bookTitle → tidak sesuai konvensi
✅ Benar Kolom DB: book_title → snake_case
❌ Salah wire:model="book_title" (snake) tapi property $bookTitle (camel)
✅ Benar wire:model="bookTitle" cocok dengan $bookTitle

🎯 Latihan: Tebak Nama yang Benar

🟢 Latihan 1 — Model ke Tabel

Laravel akan mencari tabel apa untuk Model-Model berikut?

  1. Model Employee
  2. Model BookCategory
  3. Model SalarySlip
💡 Lihat Kunci Jawaban
  1. Employee → tabel employees
  2. BookCategory → tabel book_categories (PascalCase → snake_case + plural)
  3. SalarySlip → tabel salary_slips

🟡 Latihan 2 — Livewire Naming

Kamu menjalankan perintah: php artisan make:livewire EmployeeTable

Sebutkan:

  1. Path file PHP yang dibuat
  2. Path file Blade yang dibuat
  3. Tag yang dipakai untuk memanggilnya di Blade
💡 Lihat Kunci Jawaban
  1. app/Livewire/EmployeeTable.php
  2. resources/views/livewire/employee-table.blade.php
  3. <livewire:employee-table />

🟣 Latihan 3 — Temukan Kesalahan

Kode berikut punya 4 kesalahan konvensi. Temukan semuanya!

// Migration
Schema::create('Book', function (Blueprint $table) {      ← ??
    $table->string('bookTitle');                           ← ??
});

// Livewire Blade
<livewire:BookList />                                  ← ??

// wire:model
<input wire:model="book_title">  // property: $bookTitle  ← ??
💡 Lihat Kunci Jawaban
  1. 'Book' → seharusnya 'books' (nama tabel: snake_case, plural)
  2. 'bookTitle' → seharusnya 'book_title' (nama kolom: snake_case)
  3. <livewire:BookList /> → seharusnya <livewire:book-list /> (tag Livewire: kebab-case)
  4. wire:model="book_title" → seharusnya wire:model="bookTitle" (harus cocok dengan nama property di class yang pakai camelCase)

📝 Rangkuman

  • Laravel bekerja "magic" karena bergantung pada konvensi — ikuti atau fitur tidak akan jalan otomatis.
  • Model PascalCase Singular → tabel snake_case Plural (otomatis).
  • Hindari nama Model bahasa Indonesia — plural-nya tidak bisa ditebak oleh Laravel.
  • Livewire punya 3 nama: class (BookList), file blade (book-list.blade.php), tag (<livewire:book-list />).
  • Kolom database → snake_case. Property PHP → camelCase. Keduanya harus konsisten di wire:model.
  • Pakai php artisan make:livewire — jangan buat file manual.
