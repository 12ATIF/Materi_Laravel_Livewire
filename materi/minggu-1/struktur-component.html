1.6 Struktur Component Livewire

📗 Minggu 1 ⏱️ 30 menit 🔧 Teori + Praktik

📌 Prasyarat

  • Sudah paham Pengenalan Livewire (cara kerja Livewire di balik layar)
  • Sudah paham Blade Template ({{ }}, @if, @foreach)
  • Sudah install Laravel + Livewire di komputer (Materi 1.3)

🎯 Tujuan Belajar

Setelah menyelesaikan materi ini, kamu akan mampu:

  • Membuat komponen Livewire baru menggunakan Artisan command
  • Memahami 2 bagian komponen: class PHP dan view Blade
  • Menjelaskan peran property, method, dan render()
  • Menampilkan komponen Livewire di halaman menggunakan tag Blade
  • Memahami lifecycle dasar komponen (mount, render)

🧩 Apa itu Component?

Di Livewire, setiap fitur interaktif dibuat sebagai "komponen". Satu komponen = satu unit yang mandiri dan bisa dipakai ulang.

🧱

Analogi: Lego

Bayangkan website-mu seperti bangunan Lego. Setiap komponen = satu blok Lego:

  • 🧱 Blok Counter — menghitung angka
  • 🧱 Blok SearchBar — kolom pencarian
  • 🧱 Blok BookList — daftar buku
  • 🧱 Blok LoginForm — form login

Setiap blok bisa dipasang, dicopot, dan dipakai ulang di halaman manapun!

🛠️ Membuat Component Baru

Buka terminal dan jalankan perintah Artisan:

Terminal 
# Buat komponen Livewire baru bernama "Counter"
php artisan make:livewire Counter
📂

Artisan Membuat 2 File Otomatis!

Perintah di atas akan membuat 2 file sekaligus:

📦 project-laravel/
├── app/Livewire/
│   └── Counter.php          ← Class PHP (LOGIKA)
└── resources/views/livewire/
    └── counter.blade.php     ← View Blade (TAMPILAN)

Dua file ini selalu berpasangan seperti otak 🧠 dan wajah 👁️ — otak berisi logika, wajah menampilkan hasilnya.

🧠 Bagian 1: Class PHP (Logika)

Ini adalah "otak" komponen — tempat menulis logika bisnis, menyimpan data, dan menangani aksi user.

app/Livewire/Counter.php 
namespace App\Livewire;

use Livewire\Component;

class Counter extends Component
{
    // ① PROPERTY — data yang bisa diakses oleh View
    public $count = 0;
    public $nama = 'Budi';

    // ② METHOD — fungsi yang bisa dipanggil dari View
    public function tambah()
    {
        $this->count++;
    }

    public function kurang()
    {
        $this->count--;
    }

    public function reset()
    {
        $this->count = 0;
    }

    // ③ RENDER — menghubungkan class ini ke View Blade
    public function render()
    {
        return view('livewire.counter');
    }
}

Penjelasan Baris per Baris

① Property (Data)

public $count = 0; dan public $nama = 'Budi';

Property = variabel yang otomatis tersinkronisasi antara class PHP dan view Blade. Ketika nilainya berubah di PHP, tampilan di browser langsung update!

⚠️ Property harus public agar bisa diakses view. Property private atau protected tidak akan terlihat di Blade.

② Method (Aksi)

public function tambah(), kurang(), reset()

Method = fungsi yang bisa dipanggil dari View menggunakan wire:click, wire:submit, dll. Inilah "aksi" yang user bisa lakukan.

③ Render (Penyambung)

public function render()

Method wajib yang mengembalikan view Blade. Laravel tahu untuk mencari file resources/views/livewire/counter.blade.php dari string 'livewire.counter'.

👁️ Bagian 2: View Blade (Tampilan)

Ini adalah "wajah" komponen — tampilan yang dilihat user di browser.

resources/views/livewire/counter.blade.php 
<!-- ⚠️ WAJIB: satu root element (div) -->
<div>
    <h2>Halo, {{ $nama }}!</h2>
    <p>Nilai counter: <strong>{{ $count }}</strong></p>

    <!-- Tombol memanggil method di class PHP -->
    <button wire:click="tambah">➕ Tambah</button>
    <button wire:click="kurang">➖ Kurang</button>
    <button wire:click="reset">🔄 Reset</button>
</div>
🔍

Cara Baca Kode Ini

  • {{ $nama }} — menampilkan property $nama dari class PHP (Budi)
  • {{ $count }} — menampilkan property $count (nilainya berubah saat tombol diklik)
  • wire:click="tambah" — saat tombol diklik, panggil method tambah() di class PHP

Perhatikan: property dan method yang ada di class PHP langsung bisa diakses di view Blade!

📄 Menampilkan Component di Halaman

Setelah komponen dibuat, bagaimana menampilkannya? Ada 2 cara:

Cara 1: Tag Livewire (Direkomendasikan)

resources/views/welcome.blade.php 
@extends('layouts.app')

@section('content')
    <h1>Selamat Datang</h1>

    <!-- Sisipkan komponen Counter di sini -->
    <livewire:counter />
@endsection

Cara 2: Directive Blade

resources/views/welcome.blade.php 
<!-- Cara alternatif menggunakan directive -->
@livewire('counter')
💡

Mana yang Lebih Baik?

Kedua cara hasilnya sama. Tapi <livewire:counter /> lebih populer karena mirip syntax HTML biasa dan lebih mudah dibaca.

🔄 Lifecycle Component

Setiap komponen Livewire punya "siklus hidup" — fase-fase yang dilewati dari awal sampai selesai:

🏗️ mount()
inisialisasi
→
🎨 render()
tampilkan
→
👤 User Aksi
klik, ketik
→
⚙️ Method()
proses
→
🎨 render()
re-render
Method Lifecycle Kapan Dipanggil Contoh Penggunaan
mount() Sekali saat komponen pertama dimuat Set nilai awal, ambil data dari URL
boot() Setiap request (termasuk mount) Setup yang diperlukan di setiap request
render() Setiap kali ada perubahan data Mengembalikan view Blade (wajib ada)
updated($name) Setelah property berubah Validasi input real-time
updating($name) Sebelum property berubah Mencegah perubahan yang tidak valid
hydrate() Setelah komponen di-refresh dari snapshot Reinisialisasi object yang tidak bisa di-serialize
dehydrate() Sebelum komponen disimpan ke snapshot Cleanup sebelum response dikirim
📢

Catatan Guru RPL

Untuk saat ini, kamu cukup menguasai mount(), render(), dan updated(). Tiga method ini sudah mencakup 90% kebutuhan development. Hook lainnya (boot, hydrate, dehydrate) akan kamu perlukan saat membuat aplikasi yang lebih kompleks nanti — kita bahas detail di materi Lifecycle & Events.

Contoh mount() — Inisialisasi Data

app/Livewire/Greeting.php 
class Greeting extends Component
{
    public $nama;
    public $waktu;

    // mount() dipanggil SEKALI saat komponen pertama dimuat
    public function mount($nama)
    {
        $this->nama = $nama;
        $hour = date('H');

        if ($hour < 12) {
            $this->waktu = 'Pagi';
        } elseif ($hour < 17) {
            $this->waktu = 'Siang';
        } else {
            $this->waktu = 'Malam';
        }
    }

    public function render()
    {
        return view('livewire.greeting');
    }
}
Cara pakai di halaman 
<!-- Kirim parameter ke mount() -->
<livewire:greeting nama="Budi" />

<!-- Hasil di browser: "Selamat Pagi, Budi!" -->

🧮 Computed Properties (Baru!)

Kadang kamu butuh property yang nilainya dihitung dari property lain. Misalnya, menampilkan "total harga" yang dihitung dari array belanja. Untuk ini, Livewire punya fitur Computed Properties.

app/Livewire/KeranjangBelanja.php 
use Livewire\Attributes\Computed;

class KeranjangBelanja extends Component
{
    public $items = [
        ['nama' => 'Buku PHP', 'harga' => 75000],
        ['nama' => 'Buku Laravel', 'harga' => 95000],
    ];

    // Computed Property — dihitung otomatis!
    #[Computed]
    public function totalHarga()
    {
        return collect($this->items)->sum('harga');
    }

    #[Computed]
    public function jumlahItem()
    {
        return count($this->items);
    }
}
resources/views/livewire/keranjang-belanja.blade.php 
<div>
    <h3>🛒 Keranjang ({{ $this->jumlahItem }} item)</h3>
    <p>Total: Rp {{ number_format($this->totalHarga) }}</p>

    <!-- Perhatikan: pakai $this-> bukan $totalHarga -->
</div>
⚠️

Perbedaan Penting!

  • Property biasa → akses dengan {{ $nama }}
  • Computed property → akses dengan {{ $this->totalHarga }} (pakai $this->)

Computed property di-cache per render, jadi kalau dipanggil 3x di view, method-nya hanya dijalankan 1x. Hemat performa!

📤 Mengirim Data Tambahan dari render()

Selain property, kamu juga bisa mengirim data dari render() ke view, mirip seperti controller di Laravel biasa:

app/Livewire/DaftarSiswa.php 
public function render()
{
    // Kirim data tambahan ke view
    return view('livewire.daftar-siswa', [
        'siswa' => Siswa::orderBy('nama')->get(),
        'totalSiswa' => Siswa::count(),
    ]);
}
💡

Kapan Pakai Apa?

  • Property public → untuk data yang bisa diubah oleh user (input form, counter, dll)
  • Data dari render() → untuk data yang hanya ditampilkan (daftar dari database, statistik)
  • Computed property → untuk kalkulasi dari property lain (total harga, rata-rata nilai)

⏳ Lazy Components (Pengenalan)

Kalau halaman punya banyak komponen Livewire, loading bisa lambat. Dengan Lazy Components, komponen bisa di-load setelah halaman selesai dimuat:

app/Livewire/DaftarSiswa.php 
use Livewire\Attributes\Lazy;

#[Lazy]  // Tambahkan ini saja!
class DaftarSiswa extends Component
{
    // ... kode seperti biasa
}
Di halaman Blade 
<!-- Komponen ini akan muncul setelah halaman selesai dimuat -->
<livewire:daftar-siswa />

<!-- Saat loading, akan tampil placeholder -->
<!-- Kamu bisa custom placeholder di method placeholder() -->
📝

Catatan Guru RPL

Lazy Components berguna untuk halaman yang punya banyak komponen berat (misalnya: dashboard dengan chart + tabel + notifikasi). Untuk saat ini cukup tahu konsepnya saja, nanti praktikkan di materi lanjutan.

🗺️ Peta Hubungan: Class ↔ View

Ini adalah inti dari komponen Livewire — bagaimana class dan view saling terhubung:

🧠 Class PHP

public $count = 0;

public $nama;

function tambah()

function render()

⟷

👁️ View Blade

{{ $count }}

{{ $nama }}

wire:click="tambah"

→ counter.blade.php

🔑

Aturan Emas Livewire

  1. Property public di class → otomatis tersedia di view sebagai $variabel
  2. Method public di class → bisa dipanggil dari view lewat wire:click, wire:submit, dll
  3. render() → wajib ada, menghubungkan class ke file Blade-nya
  4. View Livewire → HARUS punya satu root element <div>

⚠️ Kesalahan Umum Pemula

❌ Tidak ada satu root element di view

View Livewire HARUS dibungkus dalam satu element. Kalau ada dua element sejajar tanpa wrapper, akan error.

✅ Solusi: Selalu bungkus konten view dalam <div>...</div>.

❌ Property dengan tipe data yang tidak didukung

Livewire hanya mendukung tipe data sederhana: string, int, float, boolean, array, dan model Eloquent. Jangan simpan object custom atau closure.

✅ Solusi: Gunakan tipe data sederhana. Kalau butuh object, konversi dulu ke array.

❌ Lupa method render()

Tanpa render(), Livewire tidak tahu view mana yang harus ditampilkan.

✅ Solusi: Method render() wajib ada dan harus return view('livewire.nama-komponen').

❌ Nama class dan nama file tidak cocok

Class Counter harus ada di file Counter.php, view-nya di counter.blade.php (huruf kecil).

✅ Solusi: Gunakan perintah php artisan make:livewire NamaKomponen agar Laravel yang buatkan dengan penamaan yang benar!

📋 Ringkasan Materi

  • Setiap komponen Livewire terdiri dari 2 file: Class PHP (logika) + View Blade (tampilan)
  • Buat komponen: php artisan make:livewire NamaKomponen
  • Property (public) = data yang tersinkronisasi antara PHP dan browser
  • Method (public) = aksi yang bisa dipanggil user via wire:click
  • render() = method wajib yang menghubungkan class ke view
  • mount() = inisialisasi yang dijalankan sekali saat komponen dimuat
  • Tampilkan komponen: <livewire:nama-komponen />
