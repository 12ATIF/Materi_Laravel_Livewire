1.4 Blade Template

📗 Minggu 1 ⏱️ 25 menit 📖 Teori + Praktik

📌 Prasyarat

  • Sudah menginstall Laravel & Livewire (Materi 1.3)
  • Paham HTML dasar: tag, atribut, struktur halaman
  • Paham PHP dasar: variabel, echo, if-else, foreach

🎯 Tujuan Belajar

Setelah menyelesaikan materi ini, kamu akan mampu:

  • Menjelaskan apa itu template engine dan kenapa Blade lebih baik dari PHP biasa
  • Menampilkan data menggunakan {{ }} dan {!! !!}
  • Menggunakan direktif Blade: @if, @foreach, @forelse, @include
  • Membuat layout yang bisa dipakai ulang di banyak halaman

💡 Apa itu Template Engine?

Kamu sudah tahu cara menampilkan data PHP di HTML. Tapi cara lama sangat berantakan:

🎨

Analogi: Lukisan vs Template

Bayangkan kamu bikin undangan. Ada dua cara:

  • Cara manual — Tulis satu per satu untuk setiap tamu. Lambat dan rawan salah.
  • Template — Buat desain 1x, masukkan data (nama, tanggal, dll) di tempat yang sudah ditandai.

Blade = template engine. Kamu desain halaman HTML sekali, lalu "isi" datanya dari Controller/database.

PHP Biasa vs Blade

Aspek PHP Biasa Blade Laravel
Tampilkan variabel <?php echo $nama; ?> {{ $nama }}
Kondisi if <?php if($x): ?> ... <?php endif; ?> @if($x) ... @endif
Loop foreach <?php foreach($arr as $i): ?> @foreach($arr as $i)
Keamanan ❌ Harus manual escape HTML ✅ Otomatis escape (aman XSS)
Readability ❌ Tanda <?php ... ?> mengganggu ✅ Bersih dan mudah dibaca

📤 Menampilkan Data dengan Blade

1. Escaped Output — {{ }}

Cara paling umum dan paling aman untuk menampilkan data:

welcome.blade.php 
<!-- Tampilkan variabel dari Controller -->
<h1>Halo, {{ $nama }}!</h1>
<p>Kamu kelas {{ $kelas }}</p>

<!-- Bisa juga ekspresi PHP -->
<p>Nama huruf besar: {{ strtoupper($nama) }}</p>
<p>Tahun sekarang: {{ date('Y') }}</p>
🛡️

Kenapa "Escaped"?

{{ }} secara otomatis mengubah karakter berbahaya menjadi aman. Contoh:

$nama = "<script>alert('hack')</script>";

Kalau pakai <?php echo $nama; ?> — JavaScript-nya akan dieksekusi (bahaya! disebut XSS attack).

Kalau pakai {{ $nama }} — akan ditampilkan sebagai teks biasa (aman!).

2. Unescaped Output — {!! !!}

Kalau kamu sengaja ingin menampilkan HTML mentah (misalnya dari WYSIWYG editor):

artikel.blade.php 
<!-- $konten = "<b>Teks tebal</b> dan <i>miring</i>" -->

<!-- Escaped: akan tampil sebagai teks "<b>Teks tebal</b>" -->
<div>{{ $konten }}</div>

<!-- Unescaped: akan tampil sebagai Teks tebal dan miring -->
<div>{!! $konten !!}</div>
⚠️

Hati-hati dengan {!! !!}

Jangan pernah pakai {!! !!} untuk data yang berasal dari input user! Hanya gunakan untuk data yang kamu kontrol sendiri (misalnya konten dari database yang sudah divalidasi).

🔀 Direktif Blade — Kontrol Alur

Blade punya "direktif" (perintah shortcut) yang memudahkan menulis logika di template:

@if — Kondisi

dashboard.blade.php 
@if($role == 'admin')
    <p>👑 Selamat datang, Admin!</p>
@elseif($role == 'guru')
    <p>👨‍🏫 Selamat datang, Bapak/Ibu Guru!</p>
@else
    <p>👋 Selamat datang, Siswa!</p>
@endif
🔍

Cara Baca

  • @if($role == 'admin') — "Kalau rolenya admin, tampilkan ini..."
  • @elseif — "Kalau bukan admin tapi guru, tampilkan ini..."
  • @else — "Kalau bukan dua-duanya, tampilkan ini..."
  • @endif — "Selesai kondisi." (Wajib ada!)

@foreach — Perulangan

books/index.blade.php 
<h2>Daftar Buku ({{ count($books) }} judul)</h2>

<ul>
@foreach($books as $book)
    <li>
        <strong>{{ $book->title }}</strong>
        oleh {{ $book->author }}
    </li>
@endforeach
</ul>

@forelse — Loop dengan Fallback Kosong ⭐

Ini fitur favorit! @forelse menangani kasus ketika data kosong:

books/index.blade.php 
@forelse($books as $book)
    <div class="book-card">
        <h3>{{ $book->title }}</h3>
        <p>{{ $book->author }}</p>
    </div>
@empty
    <!-- Tampil kalau $books kosong -->
    <p>📭 Belum ada buku. <a href="/books/create">Tambah buku pertama</a></p>
@endforelse
💡

Kapan Pakai @forelse?

Selalu gunakan @forelse daripada @foreach kalau datanya mungkin kosong. Lebih rapi daripada menulis @if(count($books) > 0) terpisah!

🏗️ Layout — Template yang Bisa Dipakai Ulang

Di aplikasi nyata, setiap halaman biasanya punya header, sidebar, dan footer yang sama. Masa harus copy-paste di setiap halaman?

Blade punya solusinya: Layout + @extends + @section.

🧩

Analogi: Bingkai Foto

Layout = bingkai foto. Kamu buat bingkai sekali (dengan header, sidebar, footer), lalu ganti-ganti isi foto-nya (konten halaman) sesuai kebutuhan.

Step 1: Buat Layout Utama

resources/views/layouts/app.blade.php 
<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <title>@yield('title', 'Perpustakaan SMK')</title>
    <link rel="stylesheet" href="/css/app.css">
    @livewireStyles
</head>
<body>
    <!-- Header yang sama di semua halaman -->
    <nav>
        <a href="/">🏠 Beranda</a>
        <a href="/books">📚 Buku</a>
    </nav>

    <!-- Konten berbeda di setiap halaman -->
    <main>
        @yield('content')
    </main>

    <!-- Footer yang sama di semua halaman -->
    <footer>
        <p>© 2025 Perpustakaan SMK</p>
    </footer>

    @livewireScripts
</body>
</html>
🔍

Penjelasan

  • @yield('title', 'Perpustakaan SMK') — Tempat yang bisa diisi oleh halaman anak. Kalau tidak diisi, pakai default "Perpustakaan SMK"
  • @yield('content') — Di sinilah konten setiap halaman akan muncul

Step 2: Buat Halaman Anak

resources/views/books/index.blade.php 
@extends('layouts.app')

@section('title', 'Daftar Buku')

@section('content')
    <h1>📚 Daftar Buku</h1>

    @forelse($books as $book)
        <div class="book-card">
            <h3>{{ $book->title }}</h3>
            <p>Penulis: {{ $book->author }}</p>
        </div>
    @empty
        <p>Belum ada buku.</p>
    @endforelse
@endsection
🔍

Cara Kerja

  • @extends('layouts.app') — "Halaman ini pakai layout layouts/app.blade.php"
  • @section('title', 'Daftar Buku') — Isi @yield('title') dengan "Daftar Buku"
  • @section('content') ... @endsection — Isi @yield('content') dengan konten di dalamnya

Hasil: Halaman ini akan punya header, sidebar, dan footer dari layout, plus konten daftar buku yang unik!

📋 Blade Cheatsheet

Simpan tabel ini sebagai referensi cepat:

Direktif Fungsi Contoh
{{ $var }} Tampilkan data (escaped/aman) {{ $nama }}
{!! $var !!} Tampilkan HTML mentah {!! $kontenHtml !!}
@if ... @endif Kondisi @if($admin) ... @endif
@foreach ... @endforeach Loop/Perulangan @foreach($items as $item)
@forelse ... @empty ... @endforelse Loop + fallback jika kosong @forelse($items as $item) ... @empty ...
@extends('layout') Pakai layout tertentu @extends('layouts.app')
@section ... @endsection Isi bagian layout @section('content') ... @endsection
@yield('name') Tempat yang diisi oleh @section @yield('content')
@include('partial') Masukkan file Blade lain @include('components.navbar')

⚠️ Kesalahan Umum Pemula

❌ Lupa @endif atau @endforeach

Setiap @if harus ditutup dengan @endif. Setiap @foreach harus ditutup dengan @endforeach.

✅ Solusi: Biasakan langsung tulis tag pembuka DAN penutup, baru isi di tengahnya.

❌ Pakai <?php echo ?> di file .blade.php

File Blade sudah punya {{ }} yang lebih ringkas dan aman. Jangan campur cara lama!

✅ Solusi: Di file .blade.php, selalu pakai {{ }} untuk output.

❌ Nama file tanpa .blade.php

File harus diakhiri .blade.php (bukan cuma .php) agar Blade directive-nya diproses.

✅ Contoh benar: index.blade.php, bukan index.php

📋 Ringkasan Materi

  • Blade = template engine Laravel yang lebih ringkas, bersih, dan aman dari PHP biasa
  • {{ }} = tampilkan data dengan escape otomatis (aman XSS)
  • {!! !!} = tampilkan HTML mentah (hati-hati, hanya untuk data terpercaya)
  • @if, @foreach, @forelse = direktif untuk logika di template
  • Layout = template yang bisa dipakai ulang via @extends + @section + @yield
  • File Blade harus berekstensi .blade.php

📝 Latihan

🟢 Level 1 — Ubah Sintaks

  1. Ubah kode PHP biasa ini menjadi sintaks Blade:
    <?php echo $nama; ?> → jawab: ...
  2. Ubah kode ini menjadi @if Blade:
    <?php if($umur >= 17): ?> Boleh SIM <?php endif; ?>
  3. Sebutkan 3 keunggulan Blade dibanding PHP biasa.

🔵 Level 2 — Praktik

  1. Buat file layout di resources/views/layouts/main.blade.php dengan navbar dan footer.
  2. Buat halaman about.blade.php yang @extends layout tersebut dan menampilkan informasi tentang sekolahmu.
  3. Buat halaman yang menampilkan daftar 5 mata pelajaran menggunakan @forelse.

🟣 Level 3 — Tantangan

  1. Buat layout yang menampilkan nama user yang sedang login di navbar (gunakan @if untuk cek apakah user sudah login atau belum).
  2. Kapan kamu harus pakai {!! !!} dan kapan tidak boleh? Berikan 2 contoh kasus masing-masing.
