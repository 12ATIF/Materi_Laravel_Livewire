3.3 Komponen Reusable

📕 Minggu 3 ⏱️ 25 menit 🔧 Teori + Praktik

📌 Prasyarat

  • Sudah paham Struktur Component
  • Sudah bisa buat CRUD dengan form dan tabel

🎯 Tujuan Belajar

  • Memahami kapan dan kenapa membuat komponen reusable
  • Mengirim data ke child component dengan props
  • Mengirim event dari child ke parent ($dispatch)
  • Membuat komponen UI yang bisa dipakai berulang

🤔 Kapan Butuh Komponen Reusable?

🧱

Analogi: Balok LEGO

  • 🧱 Komponen reusable = balok LEGO → bentuknya sama, bisa dipakai di banyak bangunan
  • 🏠 Halaman/form = bangunan LEGO → dirakit dari balok-balok kecil
  • 📋 Tanpa reusable = bikin semua dari awal setiap kali → capek dan rawan beda-beda

Kalau kamu menemukan pola yang berulang, itulah saatnya membuat komponen reusable:

❌ Copy-Paste (Duplikasi)

Alert sukses di halaman Buku → copy paste ke halaman Siswa → copy paste ke halaman Guru → ubah satu? Harus edit 3 tempat! 😰

✅ Komponen Reusable

Buat 1 komponen Alert → pakai di Buku, Siswa, Guru → ubah sekali, otomatis berubah di semua tempat! ✨

🔨 Contoh 1: Komponen Alert Reusable

Kita akan buat komponen Alert yang bisa dipakai di mana saja:

resources/views/components/alert.blade.php 
<!-- Blade Component (bukan Livewire!) -->
@props([
    'type' => 'info',     // info, success, warning, error
    'message' => '',
    'dismissible' => false,
])

@php
    $colors = [
        'info'    => 'background:#DBEAFE;color:#1D4ED8;border-left:4px solid #3B82F6',
        'success' => 'background:#D1FAE5;color:#065F46;border-left:4px solid #10B981',
        'warning' => 'background:#FEF3C7;color:#92400E;border-left:4px solid #F59E0B',
        'error'   => 'background:#FEE2E2;color:#991B1B;border-left:4px solid #EF4444',
    ];
@endphp

<div style="padding:1rem;border-radius:8px;margin-bottom:1rem;{{ $colors[$type] }}">
    {{ $message }}
    @if($dismissible)
        <button onclick="this.parentElement.remove()"
                style="float:right;background:none;border:none;cursor:pointer">✕</button>
    @endif
</div>
Cara pakai di halaman manapun 
<!-- Pakai di mana saja! -->
<x-alert type="success" message="Data berhasil disimpan!" :dismissible="true" />
<x-alert type="error" message="Gagal menghapus data!" />
<x-alert type="warning" message="Stok buku tinggal 3!" />
🔍

Cara Kerja @props

Konsep Penjelasan
@props(['type' => 'info']) Mendefinisikan props yang diterima + nilai default
<x-alert type="success"> Mengirim nilai ke props dari parent
:dismissible="true" Tanda : = kirim sebagai PHP value (boolean), tanpa : = string

🔨 Contoh 2: Komponen Modal Reusable

resources/views/components/modal.blade.php 
@props(['title' => 'Dialog', 'show' => false])

@if($show)
<div style="position:fixed;inset:0;background:rgba(0,0,0,0.5);
            display:flex;align-items:center;justify-content:center;z-index:50">
    <div style="background:white;border-radius:12px;padding:1.5rem;
                width:90%;max-width:500px">
        <h3 style="margin:0 0 1rem">{{ $title }}</h3>
        {{ $slot }}  <!-- ← Konten apapun bisa masuk sini -->
    </div>
</div>
@endif
Cara pakai Modal 
<!-- $showModal dikendalikan oleh Livewire -->
<x-modal title="Hapus Buku" :show="$showModal">
    <p>Yakin ingin menghapus buku ini?</p>
    <button wire:click="delete">Ya, Hapus</button>
    <button wire:click="$set('showModal', false)">Batal</button>
</x-modal>
💡

$slot — Konten Fleksibel

{{ $slot }} = tempat dimana konten anak masuk. Seperti "lubang LEGO" — kamu bisa isi apa saja di dalamnya: form, teks, tombol, tabel, dll.

🔄 Komunikasi Parent ↔ Child

Ada 2 arah komunikasi antara komponen:

⬇️
Parent → Child

Kirim data lewat props
<x-alert type="success">

⬆️
Child → Parent

Kirim event lewat $dispatch
$this->dispatch('bookSaved')

Contoh: Child mengirim event ke Parent

Child: BookForm.php → kirim event setelah simpan 
class BookForm extends Component
{
    public $title = '';

    public function save()
    {
        Book::create(['title' => $this->title]);
        $this->reset('title');

        // ⭐ Kirim event ke parent
        $this->dispatch('book-saved');
    }
}
Parent: BookManager.php → dengarkan event 
use Livewire\Attributes\On;

class BookManager extends Component
{
    // ⭐ Dengarkan event 'book-saved'
    #[On('book-saved')]
    public function refreshList()
    {
        // Render ulang otomatis → tampilkan buku baru
    }
}

📊 Blade Component vs Livewire Component

Aspek Blade Component
<x-alert>		 Livewire Component
<livewire:counter>
Interaktif? ❌ Statis (hanya render HTML) ✅ Interaktif (bisa wire:click dll)
Punya state? ❌ Tidak ada PHP state ✅ Ada property public
Kapan pakai? Tampilan: alert, card, modal, badge Fitur: form, tabel, counter
File views/components/ app/Livewire/ + views/livewire/

⚠️ Kesalahan Umum Pemula

❌ Lupa @props di Blade component

Data tidak masuk dan muncul error variable undefined.

✅ Selalu deklarasikan @props(['name' => 'default']) di baris pertama.

❌ Bingung kapan pakai Blade vs Livewire component

Membuat Livewire component untuk hal-hal statis (seperti card layout).

✅ Blade component = tampilan statis/layout. Livewire = butuh interaktivitas.

❌ Lupa tanda : saat kirim boolean/variable

dismissible="true" → diterima sebagai string "true", bukan boolean.

✅ Pakai :dismissible="true" atau :show="$showModal"

📋 Ringkasan

  • Komponen reusable = buat sekali, pakai berkali-kali
  • Blade component: @props + $slot → untuk UI statis
  • Parent → Child: kirim data via props
  • Child → Parent: kirim event via $dispatch + #[On]
  • Blade component untuk layout, Livewire component untuk interaktivitas

📝 Latihan

🟢 Level 1 — Memahami

  1. Apa bedanya Blade component dan Livewire component?
  2. Apa fungsi @props dan $slot?
  3. Kapan pakai props (parent→child) dan kapan pakai $dispatch (child→parent)?

  1. Buat Blade component <x-badge> yang menerima props color dan text. Contoh: <x-badge color="green" text="Aktif"/>
  2. Buat komponen <x-card> dengan props title dan $slot untuk konten fleksibel.
