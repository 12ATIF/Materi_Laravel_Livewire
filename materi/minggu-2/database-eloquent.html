2.3 Database & Eloquent ORM

📘 Minggu 2 ⏱️ 35 menit 🔧 Teori + Praktik

📌 Prasyarat

  • Sudah paham MVC Sederhana — database = bagian "Model"
  • Sudah install Laravel dan Laragon (Materi 1.3)
  • Laragon sudah menyalakan MySQL

🎯 Tujuan Belajar

Setelah menyelesaikan materi ini, kamu akan mampu:

  • Mengatur koneksi database di file .env
  • Membuat tabel menggunakan Migration
  • Membuat Model Eloquent dan memahami $fillable
  • Menggunakan Eloquent untuk operasi CRUD dasar
  • Menghubungkan Eloquent dengan komponen Livewire

🤔 Kenapa Butuh Database?

Selama ini, data yang kita simpan di property Livewire (seperti $todos = []) akan hilang saat halaman di-refresh. Untuk menyimpan data secara permanen, kita butuh database!

🏢

Analogi: Lemari Arsip Sekolah

  • 🗒️ Property Livewire = catatan di papan tulis → hapus saat bel pulang
  • 🗄️ Database = lemari arsip → data tersimpan permanen, bisa dicari kapan saja
  • 📋 Migration = label di setiap laci → menentukan isi setiap rak
  • 🤖 Eloquent = petugas yang mengambil/menyimpan arsip untuk kamu

1️⃣ Setup Database (.env)

Buka file .env di root project Laravel. Pastikan pengaturan database sesuai:

.env (root project) 
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=perpustakaan    # ← nama database kamu
DB_USERNAME=root
DB_PASSWORD=                 # ← kosong di Laragon
⚠️

Jangan Lupa Buat Database Dulu!

Buka HeidiSQL (bawaan Laragon) → klik kanan → Create new → Database → ketik nama "perpustakaan" → OK

Atau lewat terminal: masuk ke MySQL lalu ketik CREATE DATABASE perpustakaan;

2️⃣ Migration — Membuat Tabel

Migration = "blueprint" tabel database yang ditulis dalam PHP. Dengan migration, kamu bisa membuat, mengubah, dan menghapus tabel tanpa query SQL manual!

Langkah 1: Buat Migration + Model Sekaligus

Terminal 
# -m artinya sekaligus buat migration
php artisan make:model Book -m
📂

Perintah ini membuat 2 file:

📦 project-laravel/
├── app/Models/
│   └── Book.php                              ← Model
└── database/migrations/
    └── 2024_01_15_create_books_table.php     ← Migration

Langkah 2: Isi Migration

database/migrations/xxxx_create_books_table.php 
public function up(): void
{
    Schema::create('books', function (Blueprint $table) {
        $table->id();                          // kolom ID auto-increment
        $table->string('title');               // judul buku (VARCHAR)
        $table->string('author');              // penulis
        $table->text('description')->nullable(); // deskripsi (boleh kosong)
        $table->integer('year');               // tahun terbit
        $table->boolean('is_available')->default(true); // status tersedia
        $table->timestamps();                  // created_at & updated_at
    });
}

Tipe Kolom yang Sering Dipakai

Method Tipe SQL Contoh Data
$table->string('nama') VARCHAR(255) "Harry Potter"
$table->text('isi') TEXT (panjang) Deskripsi, konten
$table->integer('stok') INT 25, 100
$table->boolean('aktif') TINYINT(1) true / false
$table->date('tanggal') DATE 2024-01-15
$table->timestamps() 2 kolom TIMESTAMP created_at, updated_at

Langkah 3: Jalankan Migration

Terminal 
# Jalankan semua migration yang belum dijalankan
php artisan migrate

# Kalau mau reset dan jalankan ulang semua migration
php artisan migrate:fresh

3️⃣ Model — Penghubung ke Tabel

Model Eloquent = class PHP yang merepresentasikan satu tabel database. Setiap baris di tabel = satu object model.

app/Models/Book.php 
namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Book extends Model
{
    // Kolom yang boleh diisi massal (mass assignment)
    protected $fillable = [
        'title',
        'author',
        'description',
        'year',
        'is_available',
    ];
}
⚠️

Kenapa $fillable Penting?

Tanpa $fillable, Laravel akan menolak Book::create([...]) demi keamanan. Hanya kolom yang didaftarkan di $fillable yang boleh diisi lewat array.

Ini mencegah hacker menyisipkan data ke kolom yang tidak seharusnya!

Penamaan otomatis Laravel

Nama Model Nama Tabel (otomatis) Aturan
Book books Singular → Plural
User users Singular → Plural
Category categories Singular → Plural
BookCategory book_categories CamelCase → snake_case + Plural

4️⃣ Eloquent CRUD — Cheat Sheet

Eloquent membuat operasi database semudah menulis PHP biasa. Tidak perlu query SQL!

➕ CREATE (Tambah)

Book::create([
    'title' => 'Laravel',
    'author' => 'Taylor',
    'year' => 2024,
]);

👁️ READ (Baca)

Book::all();         // Semua
Book::find(1);       // By ID
Book::latest()->get(); // Terbaru
Book::where('year', 2024)
    ->get();         // Filter

✏️ UPDATE (Ubah)

$book = Book::find(1);
$book->update([
    'title' => 'Laravel 11',
    'year' => 2025,
]);

🗑️ DELETE (Hapus)

$book = Book::find(1);
$book->delete();

// Atau langsung:
Book::destroy(1);

5️⃣ Eloquent di Livewire

Sekarang kita gabungkan Eloquent dengan Livewire — data tersimpan permanen di database!

app/Livewire/BookList.php 
namespace App\Livewire;

use Livewire\Component;
use App\Models\Book;

class BookList extends Component
{
    public $title = '';
    public $author = '';

    // CREATE — Tambah buku baru ke database
    public function addBook()
    {
        Book::create([
            'title'  => $this->title,
            'author' => $this->author,
            'year'   => date('Y'),
        ]);

        // Kosongkan form setelah submit
        $this->title = '';
        $this->author = '';
    }

    // DELETE — Hapus buku dari database
    public function deleteBook($id)
    {
        Book::find($id)->delete();
    }

    // READ — Ambil semua buku, tampilkan di view
    public function render()
    {
        return view('livewire.book-list', [
            'books' => Book::latest()->get()
        ]);
    }
}
resources/views/livewire/book-list.blade.php 
<div>
    <!-- Form tambah buku -->
    <form wire:submit="addBook">
        <input wire:model="title" placeholder="Judul buku...">
        <input wire:model="author" placeholder="Nama penulis...">
        <button type="submit">➕ Tambah Buku</button>
    </form>

    <!-- Daftar buku dari database -->
    @forelse($books as $book)
        <div style="display:flex;justify-content:space-between;padding:0.75rem;border-bottom:1px solid #eee;">
            <div>
                <strong>{{ $book->title }}</strong>
                <small>oleh {{ $book->author }}</small>
            </div>
            <button wire:click="deleteBook({{ $book->id }})">🗑️</button>
        </div>
    @empty
        <p>📭 Belum ada buku. Tambahkan yang pertama!</p>
    @endforelse
</div>
✨

Sekarang Data Permanen!

Coba refresh halaman — data buku tetap ada karena tersimpan di database MySQL. Bandingkan dengan contoh Todo List di materi sebelumnya yang datanya hilang saat refresh!

⚠️ Kesalahan Umum Pemula

❌ Lupa mendaftarkan kolom di $fillable

Book::create([...]) gagal dengan error "Mass Assignment" karena kolom tidak ada di $fillable.

✅ Solusi: Pastikan semua kolom yang ingin diisi lewat create() ada di array $fillable pada model.

❌ Lupa jalankan php artisan migrate

Sudah buat migration tapi tabel belum ada di database → error "table not found".

✅ Solusi: Selalu jalankan php artisan migrate setelah membuat atau mengubah migration.

❌ Database belum dibuat di MySQL

File .env mengarah ke database "perpustakaan" tapi database belum dibuat → error koneksi.

✅ Solusi: Buat database dulu lewat HeidiSQL atau terminal sebelum menjalankan migrate.

❌ Nama model tidak sesuai konvensi

Model harus singular + PascalCase: Book (bukan Books atau book).

✅ Solusi: Gunakan php artisan make:model NamaModel -m agar Laravel yang atur penamaan.

📋 Ringkasan Materi

  • Database = penyimpanan data permanen (tidak hilang saat refresh)
  • .env = file konfigurasi koneksi database
  • Migration = blueprint tabel dalam PHP (php artisan make:model Nama -m)
  • Model = class PHP yang merepresentasikan tabel database
  • $fillable = daftar kolom yang boleh diisi massal (keamanan)
  • Eloquent CRUD: create(), all(), find(), update(), delete()
  • Di Livewire, ambil data di render() dan kirim ke view lewat parameter kedua view()

📝 Latihan

🟢 Level 1 — Memahami

  1. Apa perbedaan data di property Livewire ($todos = []) dan data di database?
  2. Sebutkan 3 langkah untuk membuat tabel baru! (petunjuk: make:model → isi migration → migrate)
  3. Kalau model bernama Student, apa nama tabel-nya otomatis di database?

🔵 Level 2 — Praktik

  1. Buat model "Student" dengan kolom: nama, kelas, jurusan, no_absen. Jalankan migration dan cek di HeidiSQL.
  2. Buat komponen Livewire "StudentList" yang menampilkan semua siswa dari database, dengan form tambah siswa baru.
  3. Tambahkan tombol hapus 🗑️ di setiap baris siswa.

🟣 Level 3 — Tantangan

  1. Tambahkan fitur filter by kelas di StudentList — user bisa memilih dropdown kelas X/XI/XII dan hanya siswa dari kelas itu yang ditampilkan.
  2. Apa yang terjadi kalau kamu lupa menambahkan kolom ke $fillable tapi tetap memakai create()? Coba dan jelaskan error-nya!
