2.4 Validasi Form

📘 Minggu 2 ⏱️ 25 menit 🔧 Teori + Praktik

📌 Prasyarat

  • Sudah paham wire:model dan wire:click / wire:submit
  • Sudah paham Database & Eloquent (create, read)
  • Mengerti HTML form dasar

🎯 Tujuan Belajar

Setelah menyelesaikan materi ini, kamu akan mampu:

  • Menambahkan validasi form di komponen Livewire menggunakan $rules
  • Menampilkan pesan error di view menggunakan @error
  • Membuat validasi real-time menggunakan updated()
  • Mengenal validation rules populer dan membuat pesan custom

🤔 Kenapa Butuh Validasi?

Bayangkan user mengisi form tapi:

📭
Nama kosong

Siapa yang mendaftar?

📧
Email "asal@"

Format salah!

🔢
Umur = -5

Hah? 😅

Tanpa validasi, data "sampah" masuk ke database. Validasi = satpam yang memeriksa data sebelum masuk!

🔒

Analogi: Satpam Sekolah

  • 🏫 Database = gedung sekolah
  • 🚶 Data dari form = tamu yang datang
  • 👮 Validasi = satpam yang memeriksa identitas
  • ✅ Data valid → boleh masuk → tersimpan di database
  • ❌ Data tidak valid → ditolak → pesan error ditampilkan

📝 Cara 1: Validasi Saat Submit

Cara paling umum — validasi semua field sekaligus saat form disubmit:

app/Livewire/ContactForm.php 
class ContactForm extends Component
{
    public $name = '';
    public $email = '';
    public $message = '';

    // === Rules validasi ===
    protected $rules = [
        'name'    => 'required|min:3',           // wajib, minimal 3 karakter
        'email'   => 'required|email',             // wajib, format email
        'message' => 'required|min:10|max:500',   // wajib, 10-500 karakter
    ];

    public function submit()
    {
        // ① Validasi — jika gagal, Livewire otomatis stop dan tampilkan error
        $this->validate();

        // ② Jika lolos validasi, simpan ke database
        Contact::create([
            'name'    => $this->name,
            'email'   => $this->email,
            'message' => $this->message,
        ]);

        // ③ Reset form dan tampilkan notifikasi
        $this->reset(['name', 'email', 'message']);
        session()->flash('success', 'Pesan berhasil dikirim!');
    }
}
resources/views/livewire/contact-form.blade.php 
<div>
    <!-- Notifikasi sukses -->

🔍

Cara Kerja @error

  • @error('name') — cek apakah ada error untuk field name
  • {{ $message }} — variabel otomatis berisi pesan error dari Laravel
  • @enderror — penutup blok error
  • Kalau tidak ada error? Blok di dalamnya tidak ditampilkan sama sekali!

⚡ Cara 2: Validasi Real-Time

Dengan validasi real-time, error muncul saat user mengetik, bukan menunggu submit!

app/Livewire/ContactForm.php — tambahan 
// Tambahkan method ini ke class ContactForm

// Dipanggil SETIAP KALI ada property yang berubah
public function updated($propertyName)
{
    // Validasi HANYA property yang baru berubah
    $this->validateOnly($propertyName);
}
💡

validate() vs validateOnly()

Method Validasi Kapan Pakai
$this->validate() Semua field sekaligus Saat submit form
$this->validateOnly('email') Satu field saja Saat input berubah (real-time)

Jangan lupa ubah wire:model menjadi wire:model.blur agar validasi dipicu saat user pindah ke input berikutnya:

View — pakai .blur untuk validasi real-time 
<!-- Validasi dipicu saat pindah ke field berikutnya -->
<input wire:model.blur="name" placeholder="Nama">
<input wire:model.blur="email" placeholder="Email">
<textarea wire:model.blur="message"></textarea>

📚 Daftar Rules Populer

Rule Fungsi Contoh
required Wajib diisi 'name' => 'required'
email Format email valid 'email' => 'required|email'
min:N Minimal N karakter 'password' => 'min:8'
max:N Maksimal N karakter 'name' => 'max:100'
numeric Harus angka 'umur' => 'numeric|min:1'
unique:table,col Unik di tabel (tidak boleh duplikat) 'email' => 'unique:users,email'
confirmed Harus sama dengan field _confirmation 'password' => 'confirmed'
string Harus berupa teks 'name' => 'string'
nullable Boleh kosong 'bio' => 'nullable|max:500'
in:a,b,c Harus salah satu dari daftar 'kelas' => 'in:X,XI,XII'
📝

Cara Membaca Rules

'email' => 'required|email|unique:users,email'

Artinya: Field email wajib diisi + harus format email + harus unik di tabel users

Tanda | (pipe) = "DAN" — semua rules harus terpenuhi!

🌐 Pesan Error dalam Bahasa Indonesia

Secara default, pesan error muncul dalam bahasa Inggris. Kamu bisa membuat pesan custom dalam Bahasa Indonesia:

app/Livewire/ContactForm.php — pesan custom 
protected $rules = [
    'name'    => 'required|min:3',
    'email'   => 'required|email',
    'message' => 'required|min:10',
];

// Pesan error custom dalam Bahasa Indonesia
protected $messages = [
    'name.required'    => 'Nama harus diisi ya! 😊',
    'name.min'         => 'Nama minimal 3 karakter.',
    'email.required'   => 'Email wajib diisi.',
    'email.email'      => 'Format email tidak valid. Contoh: budi@email.com',
    'message.required' => 'Pesan tidak boleh kosong.',
    'message.min'      => 'Pesan terlalu pendek, minimal 10 karakter.',
];
🎯

Format Pesan Custom

Format: 'namaField.namaRule' => 'Pesan dalam bahasa Indonesia'

Contoh: 'email.email' = pesan error untuk field "email" ketika rule "email" gagal.

🎨 Styling Input Error

Tambahkan visual feedback agar user tahu field mana yang bermasalah:

Input dengan border merah saat error 
<input wire:model.blur="email"
       placeholder="Email"

@error('email')
    <span style="color:#DC2626;font-size:0.85rem;display:block;margin-top:4px;">

    </span>
@enderror

⚠️ Kesalahan Umum Pemula

❌ Lupa memanggil $this->validate()

Rules sudah ditulis tapi tidak memanggil $this->validate() di method submit → data tetap tersimpan tanpa dicek!

✅ Solusi: Selalu panggil $this->validate() di baris pertama method submit.

❌ Nama property dan nama rule tidak cocok

Property $name tapi rule ditulis 'nama' => 'required' → validasi tidak berjalan.

✅ Solusi: Key di $rules harus persis sama dengan nama property.

❌ Pakai validate() di dalam updated() (tanpa Only)

$this->validate() di updated() akan memvalidasi semua field — error muncul di field yang bahkan belum disentuh user!

✅ Solusi: Di updated(), gunakan $this->validateOnly($propertyName)

📋 Ringkasan Materi

  • Validasi = memeriksa data sebelum disimpan ke database
  • Tulis rules di property $rules → panggil $this->validate()
  • Tampilkan error: @error('nama')...@enderror
  • Validasi real-time: tambahkan method updated() + validateOnly()
  • Pesan custom: property $messages dengan format 'field.rule' => 'pesan'
  • Rules populer: required, email, min, max, unique, confirmed

📝 Latihan

🟢 Level 1 — Memahami

  1. Apa perbedaan $this->validate() dan $this->validateOnly()? Kapan pakai masing-masing?
  2. Tulis rules untuk form yang punya field: nama (wajib, min 3), umur (wajib, angka, min 1), email (wajib, format email).
  3. Bagaimana cara menampilkan error di view? Tulis contoh kodenya!

🔵 Level 2 — Praktik

  1. Buat komponen "RegistrasiSiswa" dengan field: nama, NIS, kelas (dropdown X/XI/XII), email. Tambahkan validasi lengkap dan pesan error custom dalam Bahasa Indonesia.
  2. Tambahkan validasi real-time menggunakan updated() + wire:model.blur
  3. Tambahkan notifikasi "✅ Data berhasil disimpan!" setelah submit sukses.

🟣 Level 3 — Tantangan

  1. Tambahkan rule unique:students,nis ke field NIS agar tidak ada NIS yang sama. Coba isi NIS yang sudah ada — apa pesan errornya?
  2. Buat field password + konfirmasi password. Gunakan rule confirmed agar kedua password harus cocok. (Petunjuk: field kedua harus bernama password_confirmation!)
